成员函数

一般来说，类的定义分为两个部分：类声明和类成员函数定义。类成员函数定义可以在类内部，也可以在类外部。但是通常情况下，建议将类声明放在头文件 .h 文件当中，类成员函数定义放在 .cpp 文件当中。

类内部定义

默认构造函数

默认构造函数 default constructor 指不需要提供任何实参就可以调用的构造函数，包括两种形式：无参构造函数以及全缺省构造函数（所有参数都有默认值），且每个类最多可以有一个默认构造函数。

如果类没有显式地定义任何构造函数，编译器就会为我们隐式地定义一个默认构造函数，又被称为合成的默认构造函数 synthesized default constructor，其效果等同于一个函数体和参数列表都为空的默认构造函数。默认构造函数在以下两种初始化场景中会被自动调用：①默认初始化 default initialization，例如 MyClass obj; 或使用 new MyClass;，此时内置类型成员（如 int、指针等）的值是未定义的，因为内置类型成员本身若默认初始化，编译器不会为它们赋一个初始值，除非作为静态存储的变量时被初始化为 0；②值初始化 value initialization，例如 MyClass obj{}; 或使用 new MyClass();，此时内置类型会被初始化为默认值 0 或 nullptr。但是，数据成员若有类内初始化器，上述两种形式都会为成员初始化为类内初始化器中的默认值。

另外，有时候编译器不能为某些类合成默认的构造函数，比如类中有未初始化的引用类型的成员变量，或者包含一个其他类类型的成员且这个成员的类型没有默认构造函数，亦或者含有未初始化的 const 成员等等。

①引用类型 T&：引用必须在定义时绑定对象，且不可修改；
②const 成员：常量在声明后不能赋值，只能在初始化阶段设值；

在构造函数体内赋值跟使用初始化列表具有本质区别：在构造函数体内赋值，编译器会先调用成员的默认构造函数创建临时对象，然后再调用赋值运算符覆盖值，这产生了两次函数调用。而使用初始化列表，编译器直接初始化成员对象，只发生一次构造。

另外，初始化列表的执行顺序，完全取决于成员变量在类中的声明顺序，而不是初始化列表中的书写顺序。建议书写时，永远让初始化列表的顺序与类中声明的顺序严格一致，并且绝对不要用尚未初始化的成员去初始化另一个成员。

拷贝、移动构造函数

除了定义类的对象如何初始化之外，类还需要控制拷贝、移动、赋值和销毁对象时发生的行为。一个类通过定义五种特殊的成员函数来控制这些操作，包括：拷贝构造函数 copy constructor、拷贝赋值运算符 copy assignment operator、移动构造函数 move constructor、移动赋值运算符 move assignment operator 和析构函数 destructor。析构函数在下面小节讲解，其他内容在下一章节 - 类的拷贝控制中详细说明。

析构函数

析构函数 destructor 是类的一个特殊成员函数，它在对象的生命周期结束时自动调用，主要用于释放对象占用的资源（如动态内存、文件句柄、网络连接、锁等）。它是资源获取即初始化 RAII 思想的核心支柱，RAII 在智能指针和内存管理的章节介绍。析构函数的名称与类的名称相同，只是前面多了波浪号 ~，例如 ~MyClass()，其无返回类型、无参数、不能被重载，并且每个类只能有一个析构函数。当一个类未定义自己的析构函数时，编译器会为它定义一个合成析构函数 synthesized destructor，合成析构函数的函数体为空。

在一个构造函数中，成员的初始化是在函数体执行之前完成的，且按照它们在类中出现的顺序进行初始化。在一个析构函数中，首先执行函数体，然后销毁成员，成员按初始化顺序的逆序销毁。成员的销毁是隐式的，销毁时发生什么依赖于成员的类型，销毁类类型的成员会执行成员自己的析构函数，而内置类型没有析构函数，直接释放内存，当然指针成员不会自动 delete 指向的堆内存，只会销毁指针自身。

触发场景

当对象被销毁时，析构函数自动执行，常见的销毁的场景包括：
①局部变量在离开其作用域时被销毁；
②对于动态分配的对象，当对指向它的指针应用 delete 运算符时被销毁；
③当对象作为容器元素时，容器整体销毁、显式清空或删除以及容器扩容时被销毁；
④在表达式中生成的临时对象，当创建它的完整表达式结束时被销毁；
⑤具有静态存储期的对象（全局变量、static 局部变量、static 成员变量）在程序正常终止时销毁。

静态成员

静态成员是 C++ 类中一种特殊成员，它不属于某个具体对象，而是属于类本身。所有该类的对象共享同一个静态成员，静态成员包括静态数据成员和静态成员函数。

静态数据成员

静态数据成员使用 static 关键字声明，但不能在类内初始化，必须在类外进行定义（通过范围解析运算符 :: ），除非是 constexpr 或 const，C++ 17 及之后，可以使用 inline 关键字直接在类内部初始化，无需再写类外定义。如果不存在其他的初始化语句，在创建第一个对象时，所有的静态数据都会被初始化为零。

聚合类

聚合类 aggregate class 是一种特殊的类或结构体，它允许使用聚合初始化 aggregate initialization 来直接初始化其成员。聚合初始化是唯一一种绕过构造函数的初始化方式，直接在对象的内存上，按声明顺序把值“填充”给成员变量。聚合初始化也是使用 {} 或 ()（C++ 20 起支持）形式初始化的，但对非聚合类使用 {} 或 () 都会调用构造函数。一个类或结构体被视为聚合类，当且仅当它满足以下所有条件：
①没有用户提供的构造函数（=default / =delete 除外）；
②所有非静态数据成员的访问权限必须是 public，静态数据成员和成员函数不受此限制；
③不能有基类（C++ 17 后允许）或虚函数。

字面值常量类

上篇文章第五章 - 表达式与运算符 - 常量表达式 - 字面值类型提过字面值常量类 literal constant class，但那里讲的比较简单。字面值常量类是字面值类型 literal type 的一种，指满足特定条件、可以在编译期作为常量表达式使用的类类型，必须同时满足：
①类必须使用析构函数的默认定义；
②至少有一个 constexpr 构造函数（非拷贝/移动构造函数），或者该类是一个聚合类；
③所有非静态数据成员和所有基类都必须是字面值类型；
④如果一个数据成员含有类内初始值，则内置类型成员的初始值必须是一条常量表达式，或者如果成员属于某种类类型，则初始值必须使用成员自己的 constexpr 构造函数。

另外，C++ 20 之前要求 constexpr 构造函数必须初始化所有非静态数据成员，但自 C++ 20 起不再强制要求，未初始化的成员将成为未初始化的状态。C++ 20 之前，如果 constexpr 构造函数没有在初始化列表中覆盖某个成员，编译器会自动检查该类内初始值，如果有，则认为该成员已被初始化，编译通过；如果没有类内初始值，且构造函数没写，则会报错。

第九章 类的拷贝控制

拷贝控制 copy control 是 C++ 类设计中的核心概念，它定义了当对象被拷贝、移动、赋值或销毁时发生的行为。一个类通过定义五种特殊的成员函数来控制这些操作，包括：拷贝构造函数 copy constructor、拷贝赋值运算符 copy assignment operator、移动构造函数 move constructor、移动赋值运算符 move assignment operator 和析构函数 destructor。

另外，如果拷贝构造函数被声明为 explicit，要求显式调用，则拷贝初始化 MyClass b = a; 会编译失败，而可以使用直接初始化 MyClass b(a);。

合成拷贝构造函数

如果我们没有为一个类定义拷贝构造函数，编译器会为我们定义一个，称为合成拷贝构造函数 synthesized copy constructor。与合成默认构造函数不同，即使定义了其他构造函数，编译器也会合成一个拷贝构造函数。合成拷贝构造函数会将其参数的成员逐个拷贝到正在创建的对象中，对于内置类型（int、指针等）直接复制字节（浅拷贝），对于类类型成员会使用该成员的拷贝构造函数。

合成拷贝构造函数因为浅拷贝的原因，存在一定的缺陷，当类有指针成员管理动态内存时，默认拷贝构造只复制指针值，即地址，导致两个对象指向同一块堆内存，析构时会发生双重释放或悬空指针问题。所以当类需要管理动态分配的内存、文件句柄等资源时，必须自定义拷贝构造函数以实现深拷贝（而非浅拷贝），否则两个对象会共享同一资源，导致重复释放或数据损坏。

移动构造函数

在 C++ 11 之前，没有直接的方法移动对象。因此，即使不必拷贝对象的情况下，也不得不拷贝。如果对象较大，或者是对象本身要求分配内存空间，进行不必要的拷贝代价非常高。为了支持移动语义，C++ 11 引入了右值引用，右值引用在第三章 - 引用部分详细探讨过，这里就不重复说明了。为了让我们自己的类型支持移动操作，需要为其定义移动构造函数和移动赋值运算符。这两个成员类似对应的拷贝操作，但它们从给定对象“窃取”资源而不是拷贝资源，使用的正是右值引用。

类似拷贝构造函数，移动构造函数 move constructor 的第一个参数是该类类型的一个引用，只不过是右值引用 T&&，若有其他参数则必须有默认值。声明时，推荐添加 noexcept，它表示该函数不会抛出异常，有助于标准库（如 std::vector）在重新分配内存时优先使用移动构造，例如：

移动赋值运算符

与拷贝赋值运算符一样，移动赋值运算符 move assignment operator 也是一个名为 operator= 的成员函数，其参数为右值引用T&&，返回值通常为 T&（*this）。同样建议声明为 noexcept，因为移动操作不应抛出异常，这有助于标准库在重新分配内存时选择移动而非拷贝，以获得强异常安全保证。

Rule of Three / Five

三法则 rule of three 指如果类需要自定义 C++ 98/03 标准支持的拷贝构造函数、拷贝赋值运算符和析构函数中的任何一个函数，那么你几乎肯定需要自定义这三个中的全部。三法则的底层逻辑在于浅拷贝与深拷贝之间的矛盾，【书签】

五法则 rule of five 之所以变成五，是因为 C++11 引入右值引用和移动语义，带来了移动构造函数和移动赋值运算符，于是法则变为了如果类需要自定义上述五种函数的任何一个，那么你几乎肯定需要自定义所有函数。

零法则 rule of zero 指理想情况下，任何自定义类都不应该自定义析构、拷贝或移动函数，而应该使用 RAII 类（如 std::vector、std::string、std::unique_ptr）作为成员，让编译器自动生成完美的拷贝/移动控制。

default / delete

【书签】

对某些类来说，编译器将这些合成的成员定义为删除的函数：
①如果类的某个成员的析构函数是删除的或不可访问的(例如，是 private 的)，则类的合成析构函数被定义为删除的。
②如果类的某个成员的拷贝构造函数是删除的或不可访问的，则类的合成拷贝构造函数被定义为删除的。
③如果类的某个成员的析构函数是删除的或不可访问的，则类合成的拷贝构造函数也被定义为删除的。
④如果类的某个成员的拷贝赋值运算符是删除的或不可访问的，或是类有一个 const 的或引用成员，则类的合成拷贝赋值运算符被定义为删除的。
⑤如果类的某个成员的析构函数是删除的或不可访问的，或是类有一个引用成员，它没有类内初始化器，或是类有一个 const 成员，它没有类内初始化器且其类型未显式定义默认构造函数，则该类的默认构造函数被定义为删除的。

本质上,这些规则的含义是：如果一个类有数据成员不能默认构造、拷贝、复制或销毁，则对应的成员函数将被定义为删除的。
