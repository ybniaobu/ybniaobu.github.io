C++ 基础学习笔记（二）

发表于2026-06-19|更新于2026-06-24|计算机语言CPP
|总字数:1.6k|阅读时长:5分钟|浏览量:

本想通过书籍来学习 C++ 的，但是粗略看了《C++ Primer Plus》、 C++ 之父 Bjarne Stroustrup 的《C++ 程序设计语言》（The C++ Programming Language）以及《C++ 之旅》（A Tour of C++）都不是很满意，要么是废话连篇，要么是结构太乱，非常不适合初学者学习。我最终决定还是按照自己的理解的结构顺序来对 C++ 的语法进行一定的梳理，内容上主要参考了菜鸟教程的 C++ 教程 、《C++ Primer》（建议初学者看这本书）、上述书籍以及 AI。

由于是基础笔记，对一些中级或高级内容只做出简单的介绍，更深入的学习建议和实践相结合。另外，本笔记主要基于 C++ 11 标准的，但是会补充一些后续标准的内容。普遍认为现代 C++ 始于 C++ 11 及之后的版本。

成员函数

一般来说，类的定义分为两个部分：类声明和类成员函数定义。类成员函数定义可以在类内部，也可以在类外部。但是通常情况下，建议将类声明放在头文件 .h 文件当中，类成员函数定义放在 .cpp 文件当中。

类内部定义

在类内部定义的成员函数是隐式内联 inline 的。在上一章函数章节有提过 inline 的作用是把代码复制粘贴到调用处，让程序跑得更快，但是对于现在的编译器来说，是否内联优化几乎完全取决于编译器。而这里的内联，主要是解决一个函数被定义在多个 .cpp 文件当中从而导致链接时重复定义的问题。比如，如果成员函数定义在头文件 .h 里，且被多个 .cpp 文件包含，必须让它隐式内联（写在类内部）或显式加 inline（写在类外部），否则链接器会报错说函数重定义。

构造函数

构造函数 constructor 是一种特殊的成员函数，常用于初始化类对象的数据成员，同样可以在类内部或外部定义。如果没有显式定义任何构造函数，编译器会自动合成一个默认构造函数。构造函数在对象被创建时会自动调用，名称与类的名称完全相同，且没有返回类型。一个类可以有多个构造函数，即构造函数重载，只要参数列表不同。当动态分配内存时，可以通过在类名后添加括号来调用构造函数，从某种意义上说，这是对构造函数的显式调用。一个简单的带参数的构造函数示例如下：

默认构造函数

每个类最多可以有一个默认构造函数，默认构造函数要么不含任何参数，要么所有参数都有默认值。

【书签】

如果类没有显式地定义构造函数，编译器就会为我们隐式地定义一个默认构造函数 default constructor。并且，只有当类没有声明任何构造函数时，编译器才会自动地生成默认构造函数。编译器创建的构造函数又被称为合成的默认构造函数 synthesized default constructor。对于大多数类来说，若数据成员存在类内的初始值，默认构造函数会用它来初始化成员。若不存在，则默认初始化该成员。

类访问修饰符

访问修饰符 access specifier 主要是用于实现数据封装 encapsulation，C++ 提供了三种访问修饰符：
①公有 public：成员可以从类外部访问；
②私有 private：成员不能从类外部访问；
③受保护 protected：成员不能从类外部访问，但其派生类可以访问。
