C++ 基础学习笔记（一）

发表于2026-05-10|更新于2026-05-16|计算机语言CPP
|总字数:3.1k|阅读时长:10分钟|浏览量:

本想通过书籍来学习 C++ 的，但是粗略看了《C++ Primer Plus》、 C++ 之父 Bjarne Stroustrup 的《C++ 程序设计语言》（The C++ Programming Language）以及《C++ 之旅》（A Tour of C++）都不是很满意，要么是废话连篇，要么是结构太乱。我最终决定还是按照自己的理解的结构顺序来对 C++ 的语法进行一定的梳理，内容上主要参考了菜鸟教程的 C++ 教程 、上述书籍以及 AI。

本笔记主要基于 C++ 11 标准，会补充一点 C++ 20 标准的内容，因为《C++ 之旅》（第三版）是基于 C++ 20 标准的。普遍认为现代 C++ 始于 C++ 11 及之后的版本。

第一章 简介与基础

C++ 程序

C++ 融合了 3 种编程方式：① C 语言代表的过程性语言；② C++ 代表的面向对象语言，通过在 C 语言上添加类；③ C++ 模板 template 支持的泛型编程。C++ 是 C 语言的超集，任何合法的 C 程序都是合法的 C++ 程序。

C++ 程序 program 运行起来的基本步骤或要素有如下：
①源代码 source code：使用文本编辑器编写的文件，这些文件组成了程序的源代码；
②编译与目标代码 compile & object code：编译 compile 意味着运行一个程序将源代码翻译为机器码 machine language，而包含了这个翻译后程序的文件就是你的程序的目标代码 object code；
③链接 Link：将目标代码和其他代码链接起来，C++ 通常会使用库 library，库包含了一系列函数的目标代码。而链接就是将你的程序的目标代码和使用到的函数的目标代码联合一些标准启动代码生成一个可以运行的版本。包含该最终产品的文件称为可执行代码 executable code。

C++ 源代码文件通常有两种后缀 .cpp 和 .h。.h 文件又被称为头文件，存放声明（函数、类、变量、常量、宏等），而 .cpp 文件存放定义（函数实现、类成员函数体、静态变量定义等）。.h 头文件通过 #include 被源文件包含，可以使用头文件保护符（#pragma once 或 #ifndef）防止重复包含。目标文件的后缀有 .obj（Windows 下）和 .o（Linux 下）。可执行文件最常见的即 .exe（Windows 下）。

链接又分为静态链接和动态链接。静态链接是将程序用到的所有库函数的二进制代码直接复制并整合到最终的可执行文件中，只有一个独立、完整的可执行文件，静态库（.lib 文件）中的目标代码被“打包”进可执行文件。而动态链接只记录符号信息（例如函数名、参数类型等），不复制库的机器码，真正解析和加载库代码的工作推迟到程序运行时，由操作系统和动态链接器完成，并且只生成一个较小的可执行文件 + 若干个动态库文件（.dll 文件，即 Dynamic Link Library 的缩写），可执行文件中仅包含对库中函数的“引用地址占位符”。

基本语法

Hello, World

行 1：包含了头文件 iostream，它是 C++ 标准库中负责输入与输出的组件，全称为 Input/Output Stream（输入/输出流），使用 cin 或 cout 进行输入和输出就需要包含 iostream；
行 2：告诉编译器使用 std 标准库命名空间，若不使用这行代码，第 6 行需要写为 std::cout << "Hello World!\n";；
行 4：最小的 C++ 程序就是 int main() { }，每个 C++ 程序必须有且仅有一个 main() 全局函数，该函数返回给系统一个 int 值（可以显示返回 0，也可以隐式返回 0，见行 7 说明），一个非零的值代表程序失败。不是所有操作系统和运行环境会使用到该返回值：基于 Linux/Unix 的环境通常会使用它，但是 Windows 的环境极少使用它；
行 6：在屏幕上显示消息 “Hello World”，操作符 << (‘‘put to’’) 将字符串字面量 "Hello World!\n" 放入标准输出流 std::cout，cout 是 character output stream 的缩写；
行 7：终止 main() 函数，并向调用进程返回值 0。这个返回语句不是必须的，如果编译器到达 main() 函数末尾没有遇到返回语句，则默认以 return 0; 结尾。

标识符

标识符 identifier 是变量名、函数名、类名、结构体名、枚举名、命名空间名等用户定义名称的统称，必须遵循 C++ 命名规则：
①第一个字符必须是字母或下划线，不能是数字；
②后续字符可以是字母、数字或下划线；
③标识符区分大小写，因此 Manpower 和 manpower 是两个不同的标识符；
④标识符不能和关键字相同；
⑤用户代码中最好不要使用以下划线开头的标识符，尤其是全局作用域，或者避免使用任何下划线开头的标识符，以免与实现冲突。以双下划线开头的标识符和以下划线后跟大写字母开头的标识符都被 C++ 标准保留给编译器、标准库等实现使用，以单下划线开头的标识符也被保留给实现，用作全局标识符。

关键字

关键字 keyword 是语言预定义的、具有特殊含义的保留标识符，不能被用户用作变量名、函数名、类名或其他标识符名称。下面大致说明一下所有关键字（可能会不全面）：

①内置类型
void：无类型；nullptr：空指针常量，类型为 std::nullptr_t；bool：布尔型，取值 true / false；char：字符型；wchar_t：宽字符，常用于支持多语言的字符处理需求；char8_t：UTF-8 字符，C++ 20 引入；char16_t：UTF-16 字符，C++ 11 引入；char32_t：UTF-32 字符，C++ 11 引入；int：整型；short：短整型；long：长整型；long long：更长的整数；float：单精度浮点数；double：双精度浮点数（默认浮点类型）。

还有个 long double 是一种‌扩展精度的浮点数据类型，Microsoft Visual C++ (MSVC) 编译器中与 double 的表示形式完全相同，但是编译器将 long double 和 double 视为非重复类型。

还可以使用修饰符关键字 signed 或 unsigned 与上述某些类型组合，例如：unsigned char、signed char、unsigned short、unsigned int、unsigned long、unsigned long long。MSVC 还有几个特殊的整数类型，包括 __int8、__int16、__int32 和 __int64 类型，其中 __int8 数据类型与 char 类型同义，__int16 与 short 类型同义，__int32 与 int 类型同义，__int64 与 long long 类型同义。

②声明相关（类型限定、存储说明等）
auto：自动类型推导，C++ 11 引入；decltype：获取表达式的精确类型，C++ 11 引入；const：表示值不可修改；constexpr：在编译期求值，要求变量是一个编译期常量，C++ 11 引入；consteval：用于声明立即函数，要求函数必须在编译期求值，C++ 20 引入；constinit：强制变量在编译期进行常量初始化，C++ 20 引入；static：静态（用于静态全局变量、函数内保持值以及静态成员变量或函数）；extern：声明外部链接，告诉编译器变量或函数在文件中定义；volatile：表示变量可能被意外修改，禁止编译器优化；thread_local：声明线程局部变量，C++11 引入；mutable：允许一个类的非静态数据成员在 const 成员函数；register：建议编译器将变量存储在 CPU 寄存器中，C++11 被标记为弃用，在 C++17 被正式移除；

③用户自定义类型
有 class、struct、union、enum 和 enum class，后面章节会详细说明。

④类与面向对象相关
访问控制相关的有 —— public：公有成员；private：私有成员；protected：受保护成员；friend：友元，允许访问 private、protected。

函数说明相关的有 —— virtual：虚函数；explicit：防止构造函数或转换函数发生隐式类型转换；inline：建议函数内联；noexcept：声明一个函数不会抛出任何异常，C++ 11 引入；override：显式声明派生类中的虚函数正在重写基类的虚函数，C++ 11 引入；final：禁止继承或重写，C++ 11 引入。

⑤其他
内存管理相关的有 —— new：动态分配内存并构造对象；delete：析构对象并释放内存；alignas：指定变量的对齐字节数，C++ 11 引入；alignof：查询类型的对齐要求，C++ 11 引入；sizeof：获取类型或表达式所占字节数。另外，malloc / free 是 C 标准库函数，不是 C++ 关键字；

类型转换相关的有 —— static_cast：编译时进行的类型转换；dynamic_cast：运行时进行的安全向下转型；reinterpret_cast：低层次的位模式重解释；const_cast：移除或添加 const / volatile 限定符。

流程控制相关的有 —— if / else、switch / case / default、for、while、do、break、continue、return、goto。

异常处理相关的有 —— try、catch、throw。

模板泛型相关的有 —— template：模板；typename：在模板参数列表中表示类型参数或声明依赖类型；concept 和 requires： C++ 20 概念 concept 的核心。

模块相关的有 —— 都是 C++ 20 引入，export、import、module。

协程相关的有 —— 都是 C++ 20 引入，co_await：暂停协程，等待操作完成；co_yield：从协程产生一个值并暂停；co_return：从协程返回。

其他 —— typedef：为已有类型创建别名；using：命名空间管理或类型别名；static_assert：编译时断言，C++ 11 引入；typeid：运行时类型识别；asm：用于嵌入汇编语言指令，称为内联汇编 inline assembly。
