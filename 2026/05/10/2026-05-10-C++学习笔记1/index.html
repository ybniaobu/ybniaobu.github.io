第一章 简介与基础

C++ 程序

C++ 融合了 3 种编程方式：① C 语言代表的过程性语言；② C++ 代表的面向对象语言，通过在 C 语言上添加类；③ C++ 模板 template 支持的泛型编程。C++ 是 C 语言的超集，任何合法的 C 程序都是合法的 C++ 程序。

C++ 程序 program 运行起来的基本步骤或要素有如下：
①源代码 source code：使用文本编辑器编写的文件，这些文件组成了程序的源代码；
②编译与目标代码 compile & object code：编译 compile 意味着运行一个程序将源代码翻译为机器码 machine language，而包含了这个翻译后程序的文件就是你的程序的目标代码 object code；
③链接 Link：将目标代码和其他代码链接起来，C++ 通常会使用库 library，库包含了一系列函数的目标代码。而链接就是将你的程序的目标代码和使用到的函数的目标代码联合一些标准启动代码生成一个可以运行的版本。包含该最终产品的文件称为可执行代码 executable code。

C++ 源代码文件通常有两种后缀 .cpp 和 .h。.h 文件又被称为头文件，存放声明（函数、类、变量、常量、宏等），而 .cpp 文件存放定义（函数实现、类成员函数体、静态变量定义等）。.h 头文件通过 #include 被源文件包含，可以使用头文件保护符（#pragma once 或 #ifndef）防止重复包含。目标文件的后缀有 .obj（Windows 下）和 .o（Linux 下）。可执行文件最常见的即 .exe（Windows 下）。

链接又分为静态链接和动态链接。静态链接是将程序用到的所有库函数的二进制代码直接复制并整合到最终的可执行文件中，只有一个独立、完整的可执行文件，静态库（.lib 文件）中的目标代码被“打包”进可执行文件。而动态链接只记录符号信息（例如函数名、参数类型等），不复制库的机器码，真正解析和加载库代码的工作推迟到程序运行时，由操作系统和动态链接器完成，并且只生成一个较小的可执行文件 + 若干个动态库文件（.dll 文件，即 Dynamic Link Library 的缩写），可执行文件中仅包含对库中函数的“引用地址占位符”。

标识符

标识符 identifier 是变量名、函数名、类名、结构体名、枚举名、命名空间名等用户定义名称的统称，必须遵循 C++ 命名规则：
①第一个字符必须是字母或下划线，不能是数字；
②后续字符可以是字母、数字或下划线；
③标识符区分大小写，因此 Manpower 和 manpower 是两个不同的标识符；
④标识符不能和关键字相同；
⑤用户代码中最好不要使用以下划线开头的标识符，尤其是全局作用域，或者避免使用任何下划线开头的标识符，以免与实现冲突。以双下划线开头的标识符和以下划线后跟大写字母开头的标识符都被 C++ 标准保留给编译器、标准库等实现使用，以单下划线开头的标识符也被保留给实现，用作全局标识符。

关键字

关键字 keyword 是语言预定义的、具有特殊含义的保留标识符，不能被用户用作变量名、函数名、类名或其他标识符名称。下面大致说明一下所有关键字（可能会不全面）：

①内置类型
void：无类型；nullptr：空指针常量，类型为 std::nullptr_t；bool：布尔型，取值 true / false；char：字符型；wchar_t：宽字符，常用于支持多语言的字符处理需求；char8_t：UTF-8 字符，C++ 20 引入；char16_t：UTF-16 字符，C++ 11 引入；char32_t：UTF-32 字符，C++ 11 引入；int：整型；short：短整型；long：长整型；long long：更长的整数；float：单精度浮点数；double：双精度浮点数（默认浮点类型）。

还有个 long double 是一种‌扩展精度的浮点数据类型，Microsoft Visual C++ (MSVC) 编译器中与 double 的表示形式完全相同，但是编译器将 long double 和 double 视为非重复类型。

还可以使用修饰符关键字 signed 或 unsigned 与上述某些类型组合，例如：unsigned char、signed char、unsigned short、unsigned int、unsigned long、unsigned long long。MSVC 还有几个特殊的整数类型，包括 __int8、__int16、__int32 和 __int64 类型，其中 __int8 数据类型与 char 类型同义，__int16 与 short 类型同义，__int32 与 int 类型同义，__int64 与 long long 类型同义。

②声明相关（类型限定、存储说明等）
auto：自动类型推导，C++ 11 引入；decltype：获取表达式的精确类型，C++ 11 引入；const：表示值不可修改；constexpr：在编译期求值，要求变量是一个编译期常量，C++ 11 引入；consteval：用于声明立即函数，要求函数必须在编译期求值，C++ 20 引入；constinit：强制变量在编译期进行常量初始化，C++ 20 引入；static：静态（用于静态全局变量、函数内保持值以及静态成员变量或函数）；extern：声明外部链接，告诉编译器变量或函数在文件中定义；volatile：表示变量可能被意外修改，禁止编译器优化；thread_local：声明线程局部变量，C++11 引入；mutable：允许一个类的非静态数据成员在 const 成员函数；register：建议编译器将变量存储在 CPU 寄存器中，C++11 被标记为弃用，在 C++17 被正式移除；

③用户自定义类型
有 class、struct、union、enum 和 enum class，后面章节会详细说明。

④类与面向对象相关
访问控制相关的有 —— public：公有成员；private：私有成员；protected：受保护成员；friend：友元，允许访问 private、protected。

函数说明相关的有 —— virtual：虚函数；explicit：防止构造函数或转换函数发生隐式类型转换；inline：建议函数内联；noexcept：声明一个函数不会抛出任何异常，C++ 11 引入；override：显式声明派生类中的虚函数正在重写基类的虚函数，C++ 11 引入；final：禁止继承或重写，C++ 11 引入。

⑤其他
内存管理相关的有 —— new：动态分配内存并构造对象；delete：析构对象并释放内存；alignas：指定变量的对齐字节数，C++ 11 引入；alignof：查询类型的对齐要求，C++ 11 引入；sizeof：获取类型或表达式所占字节数。另外，malloc / free 是 C 标准库函数，不是 C++ 关键字；

类型转换相关的有 —— static_cast：编译时进行的类型转换；dynamic_cast：运行时进行的安全向下转型；reinterpret_cast：低层次的位模式重解释；const_cast：移除或添加 const / volatile 限定符。

流程控制相关的有 —— if / else、switch / case / default、for、while、do、break、continue、return、goto。

异常处理相关的有 —— try、catch、throw。

模板泛型相关的有 —— template：模板；typename：在模板参数列表中表示类型参数或声明依赖类型；concept 和 requires： C++ 20 概念 concept 的核心。

模块相关的有 —— 都是 C++ 20 引入，export、import、module。

协程相关的有 —— 都是 C++ 20 引入，co_await：暂停协程，等待操作完成；co_yield：从协程产生一个值并暂停；co_return：从协程返回。

其他 —— typedef：为已有类型创建别名；using：命名空间管理或类型别名；static_assert：编译时断言，C++ 11 引入；typeid：运行时类型识别；asm：用于嵌入汇编语言指令，称为内联汇编 inline assembly。

第二章 类型、变量与声明

程序通常需要存储信息，要将信息存储在计算机中，必须指出信息的存储位置、所需的内存空间以及存储的值，即声明变量 variable，操作系统会根据声明中变量的类型 type，来分配内存和决定在保留内存中存储什么。

类型

C++ 为程序员提供了内置类型 built-in type（包含基本类型 fundamental type）和用户自定义类型 user-defined type。用户自定义类型包括类 class、结构体 struct、联合 union 以及枚举 enum / enum class，在之后的章节讨论。

作用域与内存

作用域

声明将变量引入作用域 scope，变量的作用域可以分为以下几种：
①局部作用域 local scope：在函数和 lambda 内部声明的变量（局部变量）的作用域，只能在函数内部访问。局部变量在函数每次被调用时被创建，在函数执行完后被销毁；
②全局作用域 global scope：在所有函数和代码块之外声明的变量（全局变量）具有全局作用域，它们可以被程序中的任何函数访问。全局变量在程序开始时被创建，在程序结束时被销毁；
③块作用域 block scope：在代码块 block 内部声明的变量具有块作用域，它们只能在代码块内部访问。块作用域变量在代码块每次被执行时被创建，在代码块执行完后被销毁；
④类作用域 class scope：在类内部声明的变量（类成员变量）具有类作用域，它们可以被类的所有成员函数访问。类作用域变量的生命周期与类的生命周期相同；
⑤命名空间作用域 namespace scope：从声明点开始到命名空间结束，全局作用域本质上是全局命名空间（无名命名空间）；

其他补充要点：
①不同作用域的变量可以重名，但代码块中的变量会覆盖外层代码块的变量。总之，作用域小的屏蔽作用域大的；
②当局部变量被定义时，系统不会对其初始化，您必须自行对其初始化；
③全局变量和和局部变量同名时，可以通过作用域解析运算符 :: 来引用到全局变量，例如：::x = 2;（x 是一个全局变量）；
④存储在静态存储区的变量会在程序刚开始运行时就完成初始化，也是唯一的一次初始化。共有两种变量存储在静态存储区：全局变量和 static 变量，只不过和全局变量比起来，static 可以控制变量的可见范围。如果作为 static 局部变量在函数内定义，它的生存期为整个源程序，但是只能在定义该变量的函数内使用该变量。退出该函数后， 尽管该变量还继续存在，但不能使用它，例如：

内存分区

一般来说，程序运行时，代码、数据等都存放在不同的内存区域，这些内存区域从逻辑上做了划分，大概以下几个区域：栈区、堆区、全局/静态存储区、常量区和代码区。

①栈区 stack，也称为自动存储区 automatic storage，由编译器自动分配和释放，存储局部变量、函数参数、函数调用返回地址等。变量在其作用域内有效，离开作用域立即自动销毁。栈的特点是访问速度快，连续内存，地址由高到低增长，通常只有几 MB 大小，取决于系统或编译器设置，如果超出会造成栈溢出 stack overflow 错误。

②堆区 heap，也称为动态存储区 dynamic storage 或自由存储区 free store，由程序员手动分配和释放，使用 new / delete 或 malloc / free，也可通过智能指针（如 std::unique_ptr、std::shared_ptr）辅助自动释放。如果不释放，程序结束前会一直占用，即所谓的内存泄漏 memory leak。堆空间远大于栈，取决于系统可用内存，但访问速度相对栈慢很多。

③全局/静态存储区 Global/Static Storage Area，由编译器在程序启动时分配，在程序结束时释放，存储全局变量和静态变量。通常分为已初始化区域（.data 段）和未初始化区域（.bss 段）。

④常量存储区 Constant Storage Area，由编译器管理，通常位于只读数据段（.rodata 段），存储字符串字面量和 const 常量。C++ 中的 const 局部变量不一定在只读区（可能在栈上），只有全局或静态 const 更可能放入只读区。

⑤代码区 Code Segment，由操作系统加载可执行文件时载入，通常为只读，存储编译后的函数二进制代码（程序的机器指令）。

指针存储的是数值表示的地址，可以对指针执行算术运算和比较，包括 ++、--、+ 和 -，以及 <、>、<=、>=、==、!= ：
①加法运算：当一个指针 p 加上一个整数 n 时，结果是指针 p 向前移动 n 个元素的大小。例如，如果 p 是一个 int 类型的指针，每个 int 占 4 个字节，那么 p + n 的指针数值将增加 4n，指向当前元素后的第 n 个元素；
②减法运算：当一个指针 p 减去一个整数 n 时，结果是指针 p 向后移动 n 个元素的大小。例如，如果 p 是一个 int 类型的指针，每个 int 占4个字节，那么 p - n 的指针数值将减少 4n，指向当前元素前的第 n 个元素；
③指针自增/自减：p++ 或 ++p 等效于 p = p + 1，p-- 等效于 p = p - 1；
④指针相减：当从一个指针 p 减去另一个指针 q 时，结果是两个指针之间的元素个数。例如，如果 p 和 q 是两个 int 类型的指针，每个 int 占 4 个字节，那么 p - q 将得到两个指针之间的元素个数；

⑥关系比较：即 <、>、<=、>=，用于确定一个指针是否指向的位置在另一个指针之前或之后。关系比较在同一数组的元素之间进行是有意义的。如果指针不属于同一个数组，关系比较的结果是未定义的。

第五章 表达式与运算符

表达式基础概念

C++ 中的表达式 expression 是由一个或多个操作数 operands（变量、常量或其它表达式）和运算符 operators（如 +、-、*、/、=、==、&& 等）按语法规则组合而成的计算单元。每个表达式都有两个基本属性：
①类型 type：表达式计算结果的类型，如 int、double、bool、指针等；
②值类别 value category：每个表达式都有一个值类别，比如左值 lvalue 或右值 rvalue，决定表达式能否出现在赋值左边等位置。

值类别

在 C++ 11 之前，值类别 value category 被简单分为左值 lvalue 和右值 rvalue。左值 lvalue 代表一个有名字、有内存地址的对象，可以出现在赋值号的左边或右边；右值 rvalue 代表一个临时的、无地址的值，通常只能出现在赋值号的右边。但随着移动语义被引入到 C++ 11 之中后，问题出现了，std::move(x) 融合了左值的一些特征和右值的一些特征，它仍然明确指向那个已经存在的对象 x，可以通过它访问对象的成员，但是不能对它取地址，并且可以绑定到右值引用。于是，值类别被重新进行了定义，以区别表达式的两种独立的性质：
①身份 identity：是否指代一个确定的对象、函数；
②可移动 movable：其资源是否可以安全地被“窃取”（通常意味着该对象即将被销毁或用户明确标记为可移动）。

根据上述两个属性，所有表达式被划分为三种基本类别和两种混合类别，依次为（前三个为基本类别，后两个为混合类别）：
①左值 lvalue 表达式：有身份，不可移动的表达式，例如变量名、返回左值的函数、字符串字面量等；
②将亡值 xvalue 表达式（expiring value）：有身份，但可移动的表达式，例如返回类型右值引用的函数 std::move(x)、下标表达式、对象成员表达式等；
③纯右值 prvalue 表达式：不拥有身份且可被移动的表达式，例如除了字符串字面量之外的字面量、返回非引用的函数调用、lambda 表达式、算术表达式等；
④泛左值 glvalue 表达式：拥有身份的表达式，左值和将亡值都是泛左值；
⑤右值 rvalue 表达式：可被移动的表达式，纯右值和将亡值都是右值。

运算符

运算符又可以分为一元运算符 unary operator 和二元运算符 binary operator。作用于一个运算对象的运算符是一元运算符，如取地址运算符 & 和解引用运算符 *。作用于两个运算对象的运算符是二元运算符，如相等运算符 == 和乘法运算符 *。一些符号既能作为一元运算符也能作为二元运算符，比如 *，一个符号到底是一元运算符还是二元运算符由它的上下文决定。对于这类符号来说，它的两种用法互不相干，完全可以当成两个不同的符号。除此之外，还有一个作用于三个运算对象的三元运算符 ternary operator，即 condition ? expr1 : expr2。

switch 语句

switch 语句的基本形式如下： 

switch (expression) {
    case constant-expression:
        statement
        break;
    case constant-expression:
        statement
        break;
    ...
    default:
        statement
        break;
}

①switch 语句首先对括号里的表达式求值，表达式的值可以是整形或枚举类型。如果表达式和某个 case 标签的值匹配成功，程序从该标签之后的第一条语句开始执行，直到到达了 switch 的结尾或者是遇到一条 break 语句为止。如果 switch 语句的表达式和所有 case 都没有匹配上，将直接跳转到 switch 结构之后的第一条语句；
②case 标签必须是整形常量表达式，包括 int、char 常量或枚举值等；
③不是每一个 case 都需要包含 break。如果 case 语句不包含 break，控制流将会继续后续的 case，直到遇到 break 为止。所以我们可以故意省略掉 break 语句，使得程序能够连续执行若干个 case 标签；
④switch 语句可以有一个可选的 default 标签，如果没有任何一个 case 标签能匹配上 switch 表达式的值，程序将执行紧跟在 default 标签后面的语句。

循环语句

for 语句

for 语句的语法形式是： 

for (init-statement; condition; expression)
    statement

init-statement 必须是以下三种形式中的一种：声明语句、表达式语句或者空语句。init-statement、condition 和 expression 中的任何一个都可以被省略（使用空语句）。for 循环的执行顺序如下：
①init-statement 会首先被执行，且只会执行一次，通常这里会声明并初始化循环控制变量；
②接下来，会判断 condition。如果为真，则执行循环主体。如果为假，则不执行循环主体，且控制流会跳转到紧接着 for 循环的下一条语句；
③在执行完 for 循环主体后，控制流会跳回上面的 expression 语句。该语句允许您更新循环控制变量；
④条件再次被判断。如果为真，则执行循环，这个过程会不断重复。在条件变为假时，for 循环终止。

while 语句

while 语句的语法形式是： 

while (condition)
    statement

在 while 结构中，只要 condition 的求值结果为真就一直执行 statement。condition 不能为空，如果 condition 第一次求值就得 false，statement 一次都不执行。

do while 语句

do while 语句和 while 语句非常相似，唯一的区别是，do while 语句先执行循环体后检查条件。不管条件的值如何，我们都至少执行一次循环。do while 语句的语法形式如下所示： 

do
    statement
while (condition);

在 do 语句中，求 condition 的值之前首先执行一次 statement，condition 不能为空。如果 condition 的值为假，循环终止。否则，重复循环过程。

break 语句

break 语句负责终止离它最近的 while、do while、for 或 switch 语句，并从这些语句之后的第一条语句开始继续执行。

continue 语句

continue 语句终止最近的循环中的当前迭代并立即开始下一次迭代。continue 语句只能出现在 for、while 和 do while 循环的内部，或者嵌套在此类循环里的语句或块的内部。

continue 语句中断当前的迭代，但是仍然继续执行循环。对于 while 或者 do while 语句来说，继续判断条件的值；对于传统的 for 循环来说，继续执行 for 语句头的 expression；而对于范围 for 语句来说，则是用序列中的下一个元素初始化循环控制变量。

...

标签标识符 label 独立于变量或其他标识符的名字，因此，标签标识符可以和程序中其他实体的标识符使用同一个名字而不会相互干扰。一般情况下不建议使用 goto 语句。

形参和实参

在函数定义中的参数，全称为形式参数 formal parameter，简称形参，在 C++ 标准文档中称为 parameter。而函数调用中传递进去的具体值或变量叫实际参数 actual parameter，简称实参，在 C++ 标准文档中称为 argument。函数的调用实际上完成两项工作：①用实参初始化函数对应的形参，②是将控制权转移给被调用函数。此时，主调函数 calling function 的执行被暂时中断，被调函数 called function 开始执行。所以可以说，实参是形参的初始值。同时，实参的类型必须与对应的形参类型匹配，函数有几个形参，调用时就必须提供相同数量的实参。

局部变量

形参和函数体内部定义的变量统称为局部变量 local variable，它们仅在函数的作用域内可见，同时局部变量还会隐藏在外层作用域中同名变量，这个机制在 C++ 中被称为变量遮蔽 variable shadowing。

局部变量的生命周期依赖于定义的方式。对于普通局部变量对应的对象来说，当函数的控制路径经过变量定义语句时创建该对象，当到达定义所在的块末尾时销毁它。当函数被调用时，系统会在栈顶为这个函数分配一块专属内存，称为栈帧 stack frame，普通的局部变量就连续排列在这个栈帧里。

某些时候，有必要令局部变量的生命周期贯穿函数调用及之后的时间，此时可以将局部变量定义成 static 类型。这类局部静态对象在程序的执行路径第一次经过对象定义语句时初始化，并且直到程序终止才被销毁。第二章类型、变量与声明中的作用域与内存小节中也提到过这一点。

lambda 表达式

C++ 11 提供了匿名函数对象，也称为 lambda 函数或表达式，其语法如下： 

[capture](parameters) mutable constexpr noexcept -> return_type { body };

①[capture]：捕获列表，用于捕获上下文中的变量供 lambda 函数使用，即使为空也必须写出 []；
②(parameters)：参数列表，无参可写 () 或省略；
③mutable：可选，允许在 lambda 体内修改按值捕获的副本（默认按值捕获的副本是只读的）；
④-> return_type：后置返回类型，可省略，由编译器自动推导；
⑤{ body }：函数体，内容与普通函数一样，不过除了可以使用参数之外，还可以使用所有捕获的变量。
⑥ constexpr/noexcept：可选限定符。
