# 第一章 简介与基础
## C++ 程序
C++ 融合了 3 种编程方式：① C 语言代表的**过程性语言**；② C++ 代表的**面向对象语言**，通过在 C 语言上添加类；③ C++ 模板 template 支持的**泛型**编程。C++ 是 C 语言的超集，任何合法的 C 程序都是合法的 C++ 程序。
 
C++ 程序 program 运行起来的基本步骤或要素有如下：  
①**源代码 source code**：使用文本编辑器编写的文件，这些文件组成了程序的源代码；  
②**编译与目标代码 compile & object code**：编译 compile 意味着运行一个程序将源代码翻译为**机器码 machine language**，而包含了这个翻译后程序的文件就是你的程序的**目标代码 object code**；  
③**链接 Link**：将目标代码和其他代码链接起来，C++ 通常会使用**库 library**，库包含了一系列函数的目标代码。而链接就是将你的程序的目标代码和使用到的函数的目标代码联合一些标准启动代码生成一个可以运行的版本。包含该最终产品的文件称为**可执行代码 executable code**。

C++ 源代码文件通常有两种后缀 <kbd>.cpp</kbd> 和 <kbd>.h</kbd>。<kbd>.h</kbd> 文件又被称为**头文件**，存放声明（函数、类、变量、常量、宏等），而 <kbd>.cpp</kbd> 文件存放定义（函数实现、类成员函数体、静态变量定义等）。<kbd>.h</kbd> 头文件通过 `#include` 被源文件包含，可以使用头文件保护符（`#pragma once` 或 `#ifndef`）防止重复包含。目标文件的后缀有 <kbd>.obj</kbd>（Windows 下）和 <kbd>.o</kbd>（Linux 下）。可执行文件最常见的即 <kbd>.exe</kbd>（Windows 下）。

链接又分为**静态链接**和**动态链接**。静态链接是将程序用到的所有库函数的二进制代码直接复制并整合到最终的可执行文件中，只有一个独立、完整的可执行文件，静态库（<kbd>.lib</kbd> 文件）中的目标代码被“打包”进可执行文件。而动态链接只记录符号信息（例如函数名、参数类型等），不复制库的机器码，真正解析和加载库代码的工作推迟到程序运行时，由操作系统和动态链接器完成，并且只生成一个较小的可执行文件 + 若干个动态库文件（<kbd>.dll</kbd> 文件，即 Dynamic Link Library 的缩写），可执行文件中仅包含对库中函数的“引用地址占位符”。

## 基本语法
### Hello, World
``` C++
#include <iostream>
using namespace std;

int main()
{
    cout << "Hello World!\n";
    return 0;
}
```

行 1：包含了头文件 iostream，它是 C++ 标准库中负责输入与输出的组件，全称为 Input/Output Stream（输入/输出流），使用 cin 或 cout 进行输入和输出就需要包含 iostream；  
行 2：告诉编译器使用 std 标准库命名空间，若不使用这行代码，第 6 行需要写为 `std::cout << "Hello World!\n";`；  
行 4：最小的 C++ 程序就是 `int main() { }`，每个 C++ 程序必须有且仅有一个 `main()` 全局函数，该函数返回给系统一个 int 值（可以显示返回 0，也可以隐式返回 0，见行 7 说明），一个非零的值代表程序失败。不是所有操作系统和运行环境会使用到该返回值：基于 Linux/Unix 的环境通常会使用它，但是 Windows 的环境极少使用它；  
行 6：在屏幕上显示消息 "Hello World"，操作符 `<<` (‘‘put to’’) 将字符串字面量 `"Hello World!\n"` 放入标准输出流 `std::cout`，cout 是 character output stream 的缩写；  
行 7：终止 `main()` 函数，并向调用进程返回值 0。这个返回语句不是必须的，如果编译器到达 `main()` 函数末尾没有遇到返回语句，则默认以 `return 0;` 结尾。

C++ 20 还引入了**模块 module**，旨在替代老旧的 `#include` 机制（当然仍然可以使用），来解决头文件编译慢、宏污染、命名冲突等问题。模块主要在后面的章节介绍，这里就简单介绍一下：我们可以使用 `import std;` 来替代传统写法 `#include <iostream>`：  

``` C++
import std;
using namespace std;

int main()
{
    cout << "Hello World!\n";
    return 0;
}
```

> 我使用的 Visual Studio Community 2026 直接输入上述代码会编译错误，原因是需要在 C++ 项目属性页 -> C/C++ -> 语言中修改生成 ISO C++ 23 标准库模块属性为是。

### 标识符
**标识符 identifier** 是变量名、函数名、类名、结构体名、枚举名、命名空间名等用户定义名称的统称，必须遵循 C++ 命名规则：  
①第一个字符必须是字母或下划线，不能是数字；  
②后续字符可以是字母、数字或下划线；  
③标识符区分大小写，因此 Manpower 和 manpower 是两个不同的标识符；  
④标识符不能和关键字相同；  
⑤用户代码中最好不要使用以下划线开头的标识符，尤其是全局作用域，或者避免使用任何下划线开头的标识符，以免与实现冲突。以双下划线开头的标识符和以下划线后跟大写字母开头的标识符都被 C++ 标准保留给编译器、标准库等实现使用，以单下划线开头的标识符也被保留给实现，用作全局标识符。

### 关键字
**关键字 keyword** 是语言预定义的、具有特殊含义的保留标识符，不能被用户用作变量名、函数名、类名或其他标识符名称。下面大致说明一下所有关键字（可能会不全面）：  

①内置类型  
<kbd>void</kbd>：无类型；<kbd>nullptr</kbd>：空指针常量，类型为 `std::nullptr_t`；<kbd>bool</kbd>：布尔型，取值 true / false；<kbd>char</kbd>：字符型；<kbd>wchar_t</kbd>：宽字符，常用于支持多语言的字符处理需求；<kbd>char8_t</kbd>：UTF-8 字符，C++ 20 引入；<kbd>char16_t</kbd>：UTF-16 字符，C++ 11 引入；<kbd>char32_t</kbd>：UTF-32 字符，C++ 11 引入；<kbd>int</kbd>：整型；<kbd>short</kbd>：短整型；<kbd>long</kbd>：长整型；<kbd>long long</kbd>：更长的整数；<kbd>float</kbd>：单精度浮点数；<kbd>double</kbd>：双精度浮点数（默认浮点类型）。

还有个 <kbd>long double</kbd> 是一种‌扩展精度的浮点数据类型，Microsoft Visual C++ (MSVC) 编译器中与 double 的表示形式完全相同，但是编译器将 long double 和 double 视为非重复类型。

还可以使用修饰符关键字 <kbd>signed</kbd> 或 <kbd>unsigned</kbd> 与上述某些类型组合，例如：<kbd>unsigned char</kbd>、<kbd>signed char</kbd>、<kbd>unsigned short</kbd>、<kbd>unsigned int</kbd>、<kbd>unsigned long</kbd>、<kbd>unsigned long long</kbd>。MSVC 还有几个特殊的整数类型，包括 <kbd>__int8</kbd>、<kbd>__int16</kbd>、<kbd>__int32</kbd> 和 <kbd>__int64</kbd> 类型，其中 <kbd>__int8</kbd> 数据类型与 <kbd>char</kbd> 类型同义，<kbd>__int16</kbd> 与 <kbd>short</kbd> 类型同义，<kbd>__int32</kbd> 与 <kbd>int</kbd> 类型同义，<kbd>__int64</kbd> 与 <kbd>long long</kbd> 类型同义。

②声明相关（类型限定、存储说明等）  
<kbd>auto</kbd>：自动类型推导，C++ 11 引入；<kbd>decltype</kbd>：获取表达式的精确类型，C++ 11 引入；<kbd>const</kbd>：表示值不可修改；<kbd>constexpr</kbd>：在编译期求值，要求变量是一个编译期常量，C++ 11 引入；<kbd>consteval</kbd>：用于声明立即函数，要求函数必须在编译期求值，C++ 20 引入；<kbd>constinit</kbd>：强制变量在编译期进行常量初始化，C++ 20 引入；<kbd>static</kbd>：静态（用于静态全局变量、函数内保持值以及静态成员变量或函数）；<kbd>extern</kbd>：声明外部链接，告诉编译器变量或函数在文件中定义；<kbd>volatile</kbd>：表示变量可能被意外修改，禁止编译器优化；<kbd>thread_local</kbd>：声明线程局部变量，C++11 引入；<kbd>mutable</kbd>：允许一个类的非静态数据成员在 const 成员函数；<kbd>register</kbd>：建议编译器将变量存储在 CPU 寄存器中，C++11 被标记为弃用，在 C++17 被正式移除；

③用户自定义类型  
有 <kbd>class</kbd>、<kbd>struct</kbd>、<kbd>union</kbd>、<kbd>enum</kbd> 和 <kbd>enum class</kbd>，后面章节会详细说明。  

④类与面向对象相关  
访问控制相关的有 —— <kbd>public</kbd>：公有成员；<kbd>private</kbd>：私有成员；<kbd>protected</kbd>：受保护成员；<kbd>friend</kbd>：友元，允许访问 private、protected。  

函数说明相关的有 —— <kbd>virtual</kbd>：虚函数；<kbd>explicit</kbd>：防止构造函数或转换函数发生隐式类型转换；<kbd>inline</kbd>：建议函数内联；<kbd>noexcept</kbd>：声明一个函数不会抛出任何异常，C++ 11 引入；<kbd>override</kbd>：显式声明派生类中的虚函数正在重写基类的虚函数，C++ 11 引入；<kbd>final</kbd>：禁止继承或重写，C++ 11 引入。  

⑤其他  
内存管理相关的有 —— <kbd>new</kbd>：动态分配内存并构造对象；<kbd>delete</kbd>：析构对象并释放内存；<kbd>alignas</kbd>：指定变量的对齐字节数，C++ 11 引入；<kbd>alignof</kbd>：查询类型的对齐要求，C++ 11 引入；<kbd>sizeof</kbd>：获取类型或表达式所占字节数。另外，`malloc / free` 是 C 标准库函数，不是 C++ 关键字；  

类型转换相关的有 —— <kbd>static_cast</kbd>：编译时进行的类型转换；<kbd>dynamic_cast</kbd>：运行时进行的安全向下转型；<kbd>reinterpret_cast</kbd>：低层次的位模式重解释；<kbd>const_cast</kbd>：移除或添加 const / volatile 限定符。  

流程控制相关的有 —— <kbd>if / else</kbd>、<kbd>switch / case / default</kbd>、<kbd>for</kbd>、<kbd>while</kbd>、<kbd>do</kbd>、<kbd>break</kbd>、<kbd>continue</kbd>、<kbd>return</kbd>、<kbd>goto</kbd>。  

异常处理相关的有 —— <kbd>try</kbd>、<kbd>catch</kbd>、<kbd>throw</kbd>。  

模板泛型相关的有 —— <kbd>template</kbd>：模板；<kbd>typename</kbd>：在模板参数列表中表示类型参数或声明依赖类型；<kbd>concept</kbd> 和 <kbd>requires</kbd>： C++ 20 概念 concept 的核心。  

模块相关的有 —— 都是 C++ 20 引入，<kbd>export</kbd>、<kbd>import</kbd>、<kbd>module</kbd>。  

协程相关的有 —— 都是 C++ 20 引入，<kbd>co_await</kbd>：暂停协程，等待操作完成；<kbd>co_yield</kbd>：从协程产生一个值并暂停；<kbd>co_return</kbd>：从协程返回。  

其他 —— <kbd>typedef</kbd>：为已有类型创建别名；<kbd>using</kbd>：命名空间管理或类型别名；<kbd>static_assert</kbd>：编译时断言，C++ 11 引入；<kbd>typeid</kbd>：运行时类型识别；<kbd>asm</kbd>：用于嵌入汇编语言指令，称为内联汇编 inline assembly。

程序通常需要存储信息，要将信息存储在计算机中，必须指出信息的存储位置、所需的内存空间以及存储的值，即声明**变量 variable**，操作系统会根据声明中变量的**类型 type**，来分配内存和决定在保留内存中存储什么。

### 大小与对齐
对象或类型的大小可以通过 `sizeof` 运算符获得，例如：

``` C++
    std::cout << "short: \t\t" << "所占字节数：" << sizeof(short) << std::endl;
    std::cout << "int: \t\t" << "所占字节数：" << sizeof(int) << std::endl;
    std::cout << "unsigned: \t" << "所占字节数：" << sizeof(unsigned) << std::endl;
    std::cout << "long: \t\t" << "所占字节数：" << sizeof(long) << std::endl;
    std::cout << "unsigned long: \t" << "所占字节数：" << sizeof(unsigned long) << std::endl;
    std::cout << "double: \t" << "所占字节数：" << sizeof(double) << std::endl;
    std::cout << "long double: \t" << "所占字节数：" << sizeof(long double) << std::endl;
```


对象不仅需要足够的存储空间，CPU 访问内存时，如果数据的地址是它大小的倍数（自然**对齐 alignment**），访问效率最高；未对齐的访问可能引发性能下降甚至硬件异常。编译器会根据平台默认的对齐规则安排数据，`alignof` 运算符可以查询这个默认值，例如：  

``` C++
#include <iostream>  

int main()
{
    std::cout << alignof(char)      << '\n'; // 通常 1
    std::cout << alignof(short)     << '\n'; // 通常 2
    std::cout << alignof(int)       << '\n'; // 通常 4
    std::cout << alignof(double)    << '\n'; // 通常 8
    std::cout << alignof(long long) << '\n'; // 通常 8
}
```

我们可以使用类型说明符 `alignas(T)` 指定一个类型或变量的对齐要求（必须不小于默认值，且是 2 的幂），例如：  

``` C++
struct alignas(16) OverAligned {
    int data;
};
std::cout << alignof(OverAligned) << '\n'; // 16
```


## 声明
### 声明语句
**声明 declaration** 是一条语句，指出信息的存储类型和变量名，例如：  

``` C++
int inch;
```

大多数声明也是**定义 definition**，只有定义为变量分配内存空间。<u>如果声明时进行了初始化，包括显式或隐式初始化，那该声明可以被当作是定义</u>，例如以下声明都不属于定义：  

``` C++
double sqrt(double); // 函数声明
extern int num; // 外部变量声明，只是声明了变量，具体在哪定义的，需要编译器编译的时候去找。
struct User; // 类型名称声明
```

可以在一个声明中声明多个变量，但不建议这么做，因为会增加阅读难度：  

``` C++
int x, y; // int x; int y;
int* p, y; // int* p; int y;
int x, *q; // int x; int* q;
int v[10], *pv // int v[10]; int* pv;
```

### 初始化语句
如果为变量指定了**初始化语句 initializer**，则为变量决定了初始值，初始化语句可以有 4 种写法：  

    X a1 {v};
    X a2 = {v};
    X a3 = v;
    X a4(v);

使用 `{}` 进行初始化称为**列表初始化 list initialization**，使用列表初始化不会进行**窄化类型转换 narrowing conversion**，比如从 double 到 int 和从 int 到 char，而 `=` 会进行隐式类型转换。

所有内置类型都可以省略初始化语句，此时变量会被隐式初始化为其默认值，例如（**空初始化列表**也表示使用默认值）：  

``` C++
int a; // 等同于 int a {}; , a 变为 0
double b; // 等同于 double b {}; ，b 变为 0.0
char* p; // 等同于 char* p {}; ，p 变为 nullptr
vector<int> vec; // 等同于 vector<int> vec {}; ，vec 变为空向量容器
string s; // 等同于 string s {}; ，s 变为 ""
```

用户自定义类型默认情况下不会被初始化，除非它们是带有默认构造函数的用户自定义类型，例如上面的 string 和 vector。

### auto 和 decltype
C++ 提供了两种从表达式推断类型的方式：`auto` 和 `decltype`：  

① **auto**：  
`auto` 是一个类型占位符，用于声明变量时由编译器根据其初始化表达式自动推导该变量的类型，例如：  

``` C++
auto x = 42;                // x 的类型是 int
auto pi = 3.14159;          // pi 的类型是 double
auto name = "Alice";        // name 的类型是 const char*
auto numbers = {1, 2, 3};   // numbers 的类型是 std::initializer_list<int>

int y = 10;
auto& ref = y;              // ref 的类型是 int&
const auto z = x;           // z 的类型是 const int
```

直接使用 auto 时，编译器会忽略顶层 const 和引用，只保留值类型，例如 int 引用会被推导为普通 int。想要保留引用，必须显式加上 & 符号，写成 `auto&` 形式，想要保留常量属性，需要加上 const，写成 `const auto` 形式，如果同时需要常量和引用，可以组合使用 `const auto&` 。  

数组会被推导为指针，除非使用引用形式 auto& 才能保留数组引用类型。函数名会被推导为函数指针，同样可以通过引用保留函数引用类型。  

auto 还可以推导函数返回值，例如：  

``` C++
// 函数返回类型由 return 语句推导
auto add(int a, int b) {
    return a + b;  // 返回类型是 int
}
```

使用 auto 时，不建议使用 `{}` 列表初始化，使用 {} 推导的类型是 `std::initializer_list<T>` 类型（即花括号列表 `{...}`），例如：  

``` C++
auto v {12345}; // v 的类型是 std::initializer_list<int>
```

② **decltype**：  
`decltype` 是一个类型说明符，旨在提供一种精确获取表达式类型的方法，能够完整保留表达式的所有类型信息。基本语法形式为 `decltype(expression)`，其中 expression 可以是任意有效的 C++ 表达式或标识符，例如：  

``` C++
int x = 10;
decltype(x) a; // a 的类型被推导为 int
 
const int& ref = x;
decltype(ref) b = x; // b 的类型被推导为 const int&，与 ref 的声明类型完全一致
```

如果 decltype 的参数不是一个简单的标识符，而是一个更复杂的表达式。此时，decltype 会关注表达式的**值类别 value category**，并据此决定最终的类型：  
&emsp;&emsp; - 如果表达式是**左值 lvalue**：decltype 返回该表达式的类型的**左值引用**。即，对于类型为 T 的左值表达式，decltype 推导出的类型为 `T&` ;  
&emsp;&emsp; - 如果表达式是**将亡值 xvalue**：decltype 返回该表达式的类型的**右值引用**。即，对于类型为 T 的将亡值表达式，decltype推导出的类型为 `T&&` ；  
&emsp;&emsp; - 如果表达式是**纯右值 prvalue**：decltype 返回该表达式的类型本身（非引用类型）。即，对于类型为 T 的纯右值表达式，decltype 推导出的类型为 `T` 。

> 表达式值类别的介绍详见第五章：表达式与运算符。

``` C++
// 表达式是左值
int x = 10;
decltype((x))            // 被推导为 int&，注意"有无括号"的区别，decltype((x)) 与 decltype(x) 推导出的类型是不同的
 
// 表达式是将亡值
int x = 10;
decltype(std::move(x))   // 被推导为 int&&（std::move(x) 是将亡值，左值转化为右值引用）
 
// 表达式是纯右值
int x = 10;
decltype(x + 1)          // 被推导为int（x+1 是纯右值，推导为 int，非引用）
```

`decltype(auto)` 则是 C++ 14 引入的类型占位符，它结合了 auto 的语法便利和 decltype 的精确类型推导规则。它推导出的类型与 decltype(expression) 完全一致，而不像 auto 那样会发生类型退化（去掉引用、const/volatile 限定符等）。


## 作用域与内存
### 作用域
声明将变量引入**作用域 scope**，变量的作用域可以分为以下几种：  
①**局部作用域 local scope**：在函数和 lambda 内部声明的变量（**局部变量**）的作用域，只能在函数内部访问。局部变量在函数每次被调用时被创建，在函数执行完后被销毁；  
②**全局作用域 global scope**：在所有函数和代码块之外声明的变量（**全局变量**）具有全局作用域，它们可以被程序中的任何函数访问。全局变量在程序开始时被创建，在程序结束时被销毁；  
③**块作用域 block scope**：在代码**块 block** 内部声明的变量具有块作用域，它们只能在代码块内部访问。块作用域变量在代码块每次被执行时被创建，在代码块执行完后被销毁；  
④**类作用域 class scope**：在类内部声明的变量（**类成员变量**）具有类作用域，它们可以被类的所有成员函数访问。类作用域变量的生命周期与类的生命周期相同；  
⑤**命名空间作用域 namespace scope**：从声明点开始到命名空间结束，全局作用域本质上是全局命名空间（无名命名空间）；  

其他补充要点：  
①不同作用域的变量可以重名，但代码块中的变量会覆盖外层代码块的变量。总之，作用域小的屏蔽作用域大的；  
②当局部变量被定义时，系统不会对其初始化，您必须自行对其初始化；  
③全局变量和和局部变量同名时，可以通过作用域解析运算符 `::` 来引用到全局变量，例如：`::x = 2;`（x 是一个全局变量）；  
④存储在**静态存储区**的变量会在程序刚开始运行时就完成初始化，也是唯一的一次初始化。共有两种变量存储在静态存储区：全局变量和 static 变量，只不过和全局变量比起来，static 可以控制变量的可见范围。如果作为 static 局部变量在函数内定义，它的生存期为整个源程序，但是只能在定义该变量的函数内使用该变量。退出该函数后， 尽管该变量还继续存在，但不能使用它，例如：  

``` C++
int fun()
{
    // 在第一次进入这个函数的时候，变量 count 被初始化为 10！并接着自减 1，以后每次进入该函数，count 的值是上一次函数运行之后的值，不会被再次初始化了，仅进行自减 1 的操作。
    static int count = 10;
    return count--;
}
```

> 全局变量，静态局部变量，静态全局变量都在静态存储区分配空间。非静态全局变量的作用域是整个源程序，当一个源程序由多个源文件组成时，非静态的全局变量在各个源文件中都是有效的。而**静态全局变量**则限制了其作用域， 即只在定义该变量的源文件内有效，在同一源程序的其它源文件中不能使用它。而**静态局部变量**具有局部作用域，它只被初始化一次，自从第一次被初始化直到程序运行结束都一直存在，它和全局变量的区别在于全局变量对所有的函数都是可见的，而静态局部变量只对定义自己的函数体始终可见。

### 内存分区
一般来说，程序运行时，代码、数据等都存放在不同的内存区域，这些内存区域从逻辑上做了划分，大概以下几个区域：**栈区**、**堆区**、**全局/静态存储区**、**常量区**和**代码区**。

<div align="center">  
<img src="https://files.seeusercontent.com/2026/05/21/paM9/C01_.png" width = "30%" height = "30%" alt="图1 - 内存分区"/>
</div>

①**栈区 stack**，也称为**自动存储区 automatic storage**，由编译器自动分配和释放，存储局部变量、函数参数、函数调用返回地址等。变量在其作用域内有效，离开作用域立即自动销毁。栈的特点是访问速度快，连续内存，地址由高到低增长，通常只有几 MB 大小，取决于系统或编译器设置，如果超出会造成**栈溢出 stack overflow** 错误。  

②**堆区 heap**，也称为**动态存储区 dynamic storage** 或**自由存储区 free store**，由程序员手动分配和释放，使用 `new / delete` 或 `malloc / free`，也可通过智能指针（如 `std::unique_ptr`、`std::shared_ptr`）辅助自动释放。如果不释放，程序结束前会一直占用，即所谓的**内存泄漏 memory leak**。堆空间远大于栈，取决于系统可用内存，但访问速度相对栈慢很多。  

③**全局/静态存储区 Global/Static Storage Area**，由编译器在程序启动时分配，在程序结束时释放，存储全局变量和静态变量。通常分为已初始化区域（.data 段）和未初始化区域（.bss 段）。  

④**常量存储区 Constant Storage Area**，由编译器管理，通常位于只读数据段（.rodata 段），存储字符串字面量和 const 常量。C++ 中的 const 局部变量不一定在只读区（可能在栈上），只有全局或静态 const 更可能放入只读区。  

⑤**代码区 Code Segment**，由操作系统加载可执行文件时载入，通常为只读，存储编译后的函数二进制代码（程序的机器指令）。


# 第三章 指针、数组与引用
## 指针
每一个变量都有一个内存位置，表示在内存中的一个地址，可以使用**取地址运算符** `&` 获取地址，例如：  

``` C++
#include <iostream>
 
int main ()
{
   int  var1;
   char var2[10];
   std::cout << "var1 变量的地址： " << &var1 << std::endl;
   std::cout << "var2 变量的地址： " << &var2 << std::endl;
}
```

我的编译执行结果如下：  

``` Console
var1 变量的地址： 00000052AE2FFAF4
var2 变量的地址： 00000052AE2FFB18
```

而**指针 pointer** 就是保存另一个变量地址的变量。对于 `T` 类型，`T*` 是指向 `T` 的指针类型，`T*` 类型变量保存 `T` 类型变量的地址。例如：  

``` C++
char c = 'a';
char* p1 = &c; // p 保存 c 的地址。
char *p2 = &c;
char * p3 = &c;
char*p4 = &c;
```

虽然称为 `T*` 指针类型，但是实际上 `*` 应该视作一个运算符，即**解引用 dereference 运算符**，否则无法合理解释上述的所有指针写法。总之，`&` 表示**取地址 address of**，`*` 表示**取内容 contents of**，例如：  

``` C++
#include <iostream>

int main ()
{
    int  var = 20;   // 实际变量的声明
    int  *ip;        // 指针变量的声明
    ip = &var;       // 在指针变量中存储 var 的地址
    std::cout << "Address stored in ip variable: " << ip << std::endl;
    std::cout << "Value of *ip variable: " << *ip << std::endl;
}
```

编译执行结果如下：  

``` Console
Address stored in ip variable: 00000050074FFB64
Value of *ip variable: 20
```

> 智能指针在后续章节讲解。

### void\*
`void*` 表示指向未知类型的指针。它能够存储任意类型对象的地址，但无法直接访问所指向的数据。  

任何指针类型均可隐式转换为 `void*`，但 `void*` 不能隐式转换为其他指针类型，必须显式转换，比如使用 `static_cast`。因为编译器不知道指向的数据类型，其他操作会导致编译错误，比如解引用或指针算术，要使用 `void*`，必须显式地将其转换为其他指针类型：  

``` C++
int a = 42;
void* ptr = &a; // 任意指针均可隐式转换为 void*
int* iptr = static_cast<int*>(ptr);
*ptr; // 错误，不允许解引用 void*
++ptr; // 错误，不允许对 void* 执行递增操作
```

`void*` 绕过了类型系统，使用不当极易引发未定义行为，推荐使用模板保持类型安全。

### 空指针
在 C++ 11 标准之前，空指针可以使用 `0` 或者预处理宏 `NULL`。C 语言对 NULL 的定义为 `#define NULL ((void*)0)`。但 C++ 由于不允许 `void*` 隐式转换为其他类型的指针，为了兼容性，NULL 被定义为了 `0`，即 `#define NULL 0`。这种定义在大多数情况下能勉强工作，但到函数重载和模板推导，会产生歧义错误。于是为了解决这个历史遗留问题，C++ 11 引入了 `nullptr`。`nullptr` 的类型是 `std::nullptr_t`，可以隐式转换为任何指针类型，但不能隐式转换为非布尔类型的整数。建议优先使用 `nullptr`，完全取代 `NULL` 或 `0` 表示空指针。

### 指向指针的指针
指向指针的指针是一种多级间接寻址的形式，或者说是一个指针链。当我们定义一个指向指针的指针时，第一个指针包含了第二个指针的地址，第二个指针指向包含实际值的位置。声明时在变量名前放置两个星号 `**`，例如：  

``` C++
int  var;
int  *ptr;
int  **pptr;
var = 3000;
ptr = &var; // 获取 var 的地址
pptr = &ptr; // 使用运算符 & 获取 ptr 的地址
```

上述代码种 var，ptr，*pptr 的地址都是一样的。

### 指针与 const
const 主要在第 5 章表达式和运算符中详细说明，可以跳过这里先看那边。<u>简单来说，const 在谁后面谁就不可以修改，const 在最前面则将其后移一位，二者等效。</u>

由于 const 位置的不同，const 可以修饰被指向的对象和指针本身，分别称为**常量指针 Pointer to Constant** 和**指针常量 Constant pointer**：  

①**常量指针**即指针指向的是个常量，const 限制的是指向的值，可以有两种写法：  

``` C++
const int* p;
int const* p;
```

上面的两种书写形式是等价的，指针初始化后可以修改其指向，但是无法修改对象的值：  

``` C++
int a=10, b=20;
const int *p = &a;
p = &b; // 允许修改指向
*p = 15; // 报错，不允许修改值
```

②**指针常量**即指针本身是个常量，const 限制指针不可变，写法如下：  

``` C++
int* const p;
```

指针一旦初始化不可再更改，但是其指向对象的值是可变的：  

``` C++
int a=10, b=20;
int* const p = &a;
*p = 15; // 允许修改值
p = &b; // 报错，read-only variable 'p'
```

> C++ 程序设计语言一书中建议从右向左读，`*` 读作 pointer to。`char * const cp;` 读作 cp is a const pointer to char，`const char * p;` 读作 p is a pointer to const char。

### new / delete
指针真正的用武之地在于访问动态内存/堆/自由存储区，我们可以使用 `new` 运算符在运行时动态分配内存，如果不再需要动态分配的内存空间，可以使用 `delete` 运算符，删除之前由 `new` 运算符分配的内存，例如：  

``` C++
#include <iostream>
 
int main ()
{
    double *p = new double; // 为变量请求内存
    *p = 29494.99; // 在分配的地址存储值
    std::cout << "Address stored in p: " << p << std::endl;
    std::cout << "Value of p: " << *p << std::endl;
    delete p; // 释放内存
}
```

编译执行结果如下：  

``` Console
Address stored in p: 0000028E55506DD0
Value of p: 29495
```

> 我们也可以使用 C 的标准库函数 `malloc / free` 进行动态内存分配，但是不推荐使用。因为 malloc / free 只是机械的分配一块内存，不会调用构造函数和析构函数。严格说来用 malloc 不能算是新建了一个对象，只能说是分配了一块与该类型对象匹配的内存而已，然后强行把它解释为该类型对象，它返回 `void*`，需要进行显示转换，例如：`int* p = (int*)malloc(10 * sizeof(int));`  

new / delete 的错误操作会引发一系列严重问题，以下为常见的错误：  
①**内存泄漏 memory leak**：使用 `new` 分配内存后，忘记调用 `delete`；  
②**重复释放 double deletion**：对同一块内存调用两次 delete；  
③**悬垂指针 dangling pointer**：指针原来指向有效的内存对象，但该对象已被释放（delete 或离开作用域），指针仍保留着原来的地址；  
④**野指针 wild pointer**：指针在‌声明后未被初始化，其值是随机的（栈上的垃圾数据），指向一个不确定的地址。

## 声明
函数**声明 declaration** 告诉编译器函数的名称、返回类型和参数，例如：  

``` C++
Elem∗ next_elem(); // 返回 Elem 指针
void exit(int);
double sqrt(double);
```

函数声明可以包含<u>参数的名称</u>，但除非在函数声明的同时定义函数体，否则编译器会忽略函数声明中参数的名称，例如：  

``` C++
int max(int num1, int num2);
int max(int, int);
```

<u>函数的类型</u>由它的返回值类型和参数决定，例如：  

``` C++
double get(const vector<double>& vec, int index); // 函数类型为：double(const vector<double>&,int)
```

函数可以是类成员，对于**成员函数 member function** 来说，类名也是函数类型的一部分：  

``` C++
char& String::operator[](int index); // 函数类型为：char& String::(int)
```

可以申明多个相同名称但具有不太参数类型的函数，称为**函数重载 function overloading**：  

``` C++
void print(int);
void print(double);
void print(string);
```

## 定义
函数**定义 definition** 包含了函数的函数体，例如：  

``` C++
void swap(int*, int*); // 声明

void swap(int* p, int* q) // 定义
{
    int t = *p;
    *p = *q;
    *q = t;
}
```
