# 第一章 简介与基础
## C++ 程序
C++ 融合了 3 种编程方式：① C 语言代表的**过程性语言**；② C++ 代表的**面向对象语言**，通过在 C 语言上添加类；③ C++ 模板 template 支持的**泛型**编程。C++ 是 C 语言的超集，任何合法的 C 程序都是合法的 C++ 程序。
 
C++ 程序 program 运行起来的基本步骤或要素有如下：  
①**源代码 source code**：使用文本编辑器编写的文件，这些文件组成了程序的源代码；  
②**编译与目标代码 compile & object code**：编译 compile 意味着运行一个程序将源代码翻译为**机器码 machine language**，而包含了这个翻译后程序的文件就是你的程序的**目标代码 object code**；  
③**链接 Link**：将目标代码和其他代码链接起来，C++ 通常会使用**库 library**，库包含了一系列函数的目标代码。而链接就是将你的程序的目标代码和使用到的函数的目标代码联合一些标准启动代码生成一个可以运行的版本。包含该最终产品的文件称为**可执行代码 executable code**。

C++ 源代码文件通常有两种后缀 <kbd>.cpp</kbd> 和 <kbd>.h</kbd>。<kbd>.h</kbd> 文件又被称为**头文件**，存放声明（函数、类、变量、常量、宏等），而 <kbd>.cpp</kbd> 文件存放定义（函数实现、类成员函数体、静态变量定义等）。<kbd>.h</kbd> 头文件通过 `#include` 被源文件包含，可以使用头文件保护符（`#pragma once` 或 `#ifndef`）防止重复包含。目标文件的后缀有 <kbd>.obj</kbd>（Windows 下）和 <kbd>.o</kbd>（Linux 下）。可执行文件最常见的即 <kbd>.exe</kbd>（Windows 下）。

链接又分为**静态链接**和**动态链接**。静态链接是将程序用到的所有库函数的二进制代码直接复制并整合到最终的可执行文件中，只有一个独立、完整的可执行文件，静态库（<kbd>.lib</kbd> 文件）中的目标代码被“打包”进可执行文件。而动态链接只记录符号信息（例如函数名、参数类型等），不复制库的机器码，真正解析和加载库代码的工作推迟到程序运行时，由操作系统和动态链接器完成，并且只生成一个较小的可执行文件 + 若干个动态库文件（<kbd>.dll</kbd> 文件，即 Dynamic Link Library 的缩写），可执行文件中仅包含对库中函数的“引用地址占位符”。

## 基本语法
### Hello, World
``` C++
#include <iostream>
using namespace std;

int main()
{
    cout << "Hello World!\n";
    return 0;
}
```

行 1：包含了头文件 iostream，它是 C++ 标准库中负责输入与输出的组件，全称为 Input/Output Stream（输入/输出流），使用 cin 或 cout 进行输入和输出就需要包含 iostream；  
行 2：告诉编译器使用 std 标准库命名空间，若不使用这行代码，第 6 行需要写为 `std::cout << "Hello World!\n";`；  
行 4：最小的 C++ 程序就是 `int main() { }`，每个 C++ 程序必须有且仅有一个 `main()` 全局函数，该函数返回给系统一个 int 值（可以显示返回 0，也可以隐式返回 0，见行 7 说明），一个非零的值代表程序失败。不是所有操作系统和运行环境会使用到该返回值：基于 Linux/Unix 的环境通常会使用它，但是 Windows 的环境极少使用它；  
行 6：在屏幕上显示消息 "Hello World"，操作符 `<<` (‘‘put to’’) 将字符串字面量 `"Hello World!\n"` 放入标准输出流 `std::cout`，cout 是 character output stream 的缩写；  
行 7：终止 `main()` 函数，并向调用进程返回值 0。这个返回语句不是必须的，如果编译器到达 `main()` 函数末尾没有遇到返回语句，则默认以 `return 0;` 结尾。

C++ 20 还引入了**模块 module**，旨在替代老旧的 `#include` 机制（当然仍然可以使用），来解决头文件编译慢、宏污染、命名冲突等问题。模块主要在后面的章节介绍，这里就简单介绍一下：我们可以使用 `import std;` 来替代传统写法 `#include <iostream>`：  

``` C++
import std;
using namespace std;

int main()
{
    cout << "Hello World!\n";
    return 0;
}
```

> 我使用的 Visual Studio Community 2026 直接输入上述代码会编译错误，原因是需要在 C++ 项目属性页 -> C/C++ -> 语言中修改生成 ISO C++ 23 标准库模块属性为是。

### 标识符
**标识符 identifier** 是变量名、函数名、类名、结构体名、枚举名、命名空间名等用户定义名称的统称，必须遵循 C++ 命名规则：  
①第一个字符必须是字母或下划线，不能是数字；  
②后续字符可以是字母、数字或下划线；  
③标识符区分大小写，因此 Manpower 和 manpower 是两个不同的标识符；  
④标识符不能和关键字相同；  
⑤用户代码中最好不要使用以下划线开头的标识符，尤其是全局作用域，或者避免使用任何下划线开头的标识符，以免与实现冲突。以双下划线开头的标识符和以下划线后跟大写字母开头的标识符都被 C++ 标准保留给编译器、标准库等实现使用，以单下划线开头的标识符也被保留给实现，用作全局标识符。

### 关键字
**关键字 keyword** 是语言预定义的、具有特殊含义的保留标识符，不能被用户用作变量名、函数名、类名或其他标识符名称。下面大致说明一下所有关键字（可能会不全面）：  

①内置类型  
<kbd>void</kbd>：无类型；<kbd>nullptr</kbd>：空指针常量，类型为 `std::nullptr_t`；<kbd>bool</kbd>：布尔型，取值 true / false；<kbd>char</kbd>：字符型；<kbd>wchar_t</kbd>：宽字符，常用于支持多语言的字符处理需求；<kbd>char8_t</kbd>：UTF-8 字符，C++ 20 引入；<kbd>char16_t</kbd>：UTF-16 字符，C++ 11 引入；<kbd>char32_t</kbd>：UTF-32 字符，C++ 11 引入；<kbd>int</kbd>：整型；<kbd>short</kbd>：短整型；<kbd>long</kbd>：长整型；<kbd>long long</kbd>：更长的整数；<kbd>float</kbd>：单精度浮点数；<kbd>double</kbd>：双精度浮点数（默认浮点类型）。

还有个 <kbd>long double</kbd> 是一种‌扩展精度的浮点数据类型，Microsoft Visual C++ (MSVC) 编译器中与 double 的表示形式完全相同，但是编译器将 long double 和 double 视为非重复类型。

还可以使用修饰符关键字 <kbd>signed</kbd> 或 <kbd>unsigned</kbd> 与上述某些类型组合，例如：<kbd>unsigned char</kbd>、<kbd>signed char</kbd>、<kbd>unsigned short</kbd>、<kbd>unsigned int</kbd>、<kbd>unsigned long</kbd>、<kbd>unsigned long long</kbd>。MSVC 还有几个特殊的整数类型，包括 <kbd>__int8</kbd>、<kbd>__int16</kbd>、<kbd>__int32</kbd> 和 <kbd>__int64</kbd> 类型，其中 <kbd>__int8</kbd> 数据类型与 <kbd>char</kbd> 类型同义，<kbd>__int16</kbd> 与 <kbd>short</kbd> 类型同义，<kbd>__int32</kbd> 与 <kbd>int</kbd> 类型同义，<kbd>__int64</kbd> 与 <kbd>long long</kbd> 类型同义。

②声明相关（类型限定、存储说明等）  
<kbd>auto</kbd>：自动类型推导，C++ 11 引入；<kbd>decltype</kbd>：获取表达式的精确类型，C++ 11 引入；<kbd>const</kbd>：表示值不可修改；<kbd>constexpr</kbd>：在编译期求值，要求变量是一个编译期常量，C++ 11 引入；<kbd>consteval</kbd>：用于声明立即函数，要求函数必须在编译期求值，C++ 20 引入；<kbd>constinit</kbd>：强制变量在编译期进行常量初始化，C++ 20 引入；<kbd>static</kbd>：静态（用于静态全局变量、函数内保持值以及静态成员变量或函数）；<kbd>extern</kbd>：声明外部链接，告诉编译器变量或函数在文件中定义；<kbd>volatile</kbd>：表示变量可能被意外修改，禁止编译器优化；<kbd>thread_local</kbd>：声明线程局部变量，C++11 引入；<kbd>mutable</kbd>：允许一个类的非静态数据成员在 const 成员函数；<kbd>register</kbd>：建议编译器将变量存储在 CPU 寄存器中，C++11 被标记为弃用，在 C++17 被正式移除；

③用户自定义类型  
有 <kbd>class</kbd>、<kbd>struct</kbd>、<kbd>union</kbd>、<kbd>enum</kbd> 和 <kbd>enum class</kbd>，后面章节会详细说明。  

④类与面向对象相关  
访问控制相关的有 —— <kbd>public</kbd>：公有成员；<kbd>private</kbd>：私有成员；<kbd>protected</kbd>：受保护成员；<kbd>friend</kbd>：友元，允许访问 private、protected。  

函数说明相关的有 —— <kbd>virtual</kbd>：虚函数；<kbd>explicit</kbd>：防止构造函数或转换函数发生隐式类型转换；<kbd>inline</kbd>：建议函数内联；<kbd>noexcept</kbd>：声明一个函数不会抛出任何异常，C++ 11 引入；<kbd>override</kbd>：显式声明派生类中的虚函数正在重写基类的虚函数，C++ 11 引入；<kbd>final</kbd>：禁止继承或重写，C++ 11 引入。  

⑤其他  
内存管理相关的有 —— <kbd>new</kbd>：动态分配内存并构造对象；<kbd>delete</kbd>：析构对象并释放内存；<kbd>alignas</kbd>：指定变量的对齐字节数，C++ 11 引入；<kbd>alignof</kbd>：查询类型的对齐要求，C++ 11 引入；<kbd>sizeof</kbd>：获取类型或表达式所占字节数。另外，`malloc / free` 是 C 标准库函数，不是 C++ 关键字；  

类型转换相关的有 —— <kbd>static_cast</kbd>：编译时进行的类型转换；<kbd>dynamic_cast</kbd>：运行时进行的安全向下转型；<kbd>reinterpret_cast</kbd>：低层次的位模式重解释；<kbd>const_cast</kbd>：移除或添加 const / volatile 限定符。  

流程控制相关的有 —— <kbd>if / else</kbd>、<kbd>switch / case / default</kbd>、<kbd>for</kbd>、<kbd>while</kbd>、<kbd>do</kbd>、<kbd>break</kbd>、<kbd>continue</kbd>、<kbd>return</kbd>、<kbd>goto</kbd>。  

异常处理相关的有 —— <kbd>try</kbd>、<kbd>catch</kbd>、<kbd>throw</kbd>。  

模板泛型相关的有 —— <kbd>template</kbd>：模板；<kbd>typename</kbd>：在模板参数列表中表示类型参数或声明依赖类型；<kbd>concept</kbd> 和 <kbd>requires</kbd>： C++ 20 概念 concept 的核心。  

模块相关的有 —— 都是 C++ 20 引入，<kbd>export</kbd>、<kbd>import</kbd>、<kbd>module</kbd>。  

协程相关的有 —— 都是 C++ 20 引入，<kbd>co_await</kbd>：暂停协程，等待操作完成；<kbd>co_yield</kbd>：从协程产生一个值并暂停；<kbd>co_return</kbd>：从协程返回。  

其他 —— <kbd>typedef</kbd>：为已有类型创建别名；<kbd>using</kbd>：命名空间管理或类型别名；<kbd>static_assert</kbd>：编译时断言，C++ 11 引入；<kbd>typeid</kbd>：运行时类型识别；<kbd>asm</kbd>：用于嵌入汇编语言指令，称为内联汇编 inline assembly。

程序通常需要存储信息，要将信息存储在计算机中，必须指出信息的存储位置、所需的内存空间以及存储的值，即声明**变量 variable**，操作系统会根据声明中变量的**类型 type**，来分配内存和决定在保留内存中存储什么。

### 大小与对齐
对象或类型的大小可以通过 `sizeof` 运算符获得，例如：

``` C++
    std::cout << "short: \t\t" << "所占字节数：" << sizeof(short) << std::endl;
    std::cout << "int: \t\t" << "所占字节数：" << sizeof(int) << std::endl;
    std::cout << "unsigned: \t" << "所占字节数：" << sizeof(unsigned) << std::endl;
    std::cout << "long: \t\t" << "所占字节数：" << sizeof(long) << std::endl;
    std::cout << "unsigned long: \t" << "所占字节数：" << sizeof(unsigned long) << std::endl;
    std::cout << "double: \t" << "所占字节数：" << sizeof(double) << std::endl;
    std::cout << "long double: \t" << "所占字节数：" << sizeof(long double) << std::endl;
```


对象不仅需要足够的存储空间，CPU 访问内存时，如果数据的地址是它大小的倍数（自然**对齐 alignment**），访问效率最高；未对齐的访问可能引发性能下降甚至硬件异常。编译器会根据平台默认的对齐规则安排数据，`alignof` 运算符可以查询这个默认值，例如：  

``` C++
#include <iostream>  

int main()
{
    std::cout << alignof(char)      << '\n'; // 通常 1
    std::cout << alignof(short)     << '\n'; // 通常 2
    std::cout << alignof(int)       << '\n'; // 通常 4
    std::cout << alignof(double)    << '\n'; // 通常 8
    std::cout << alignof(long long) << '\n'; // 通常 8
}
```

我们可以使用类型说明符 `alignas(T)` 指定一个类型或变量的对齐要求（必须不小于默认值，且是 2 的幂），例如：  

``` C++
struct alignas(16) OverAligned {
    int data;
};
std::cout << alignof(OverAligned) << '\n'; // 16
```


## 声明
### 声明语句
**声明 declaration** 是一条语句，指出信息的存储类型和变量名，例如：  

``` C++
int inch;
```

大多数声明也是**定义 definition**，只有定义为变量分配内存空间。<u>如果声明时进行了初始化，包括显式或隐式初始化，那该声明可以被当作是定义</u>，例如以下声明都不属于定义：  

``` C++
double sqrt(double); // 函数声明
extern int num; // 外部变量声明，只是声明了变量，具体在哪定义的，需要编译器编译的时候去找。
struct User; // 类型名称声明
```

可以在一个声明中声明多个变量，但不建议这么做，因为会增加阅读难度：  

``` C++
int x, y; // int x; int y;
int* p, y; // int* p; int y;
int x, *q; // int x; int* q;
int v[10], *pv // int v[10]; int* pv;
```

### 初始化语句
如果为变量指定了**初始化语句 initializer**，则为变量决定了初始值，初始化语句可以有 4 种写法：  

    X a1 {v};
    X a2 = {v};
    X a3 = v;
    X a4(v);

使用 `{}` 进行初始化称为**列表初始化 list initialization**，使用列表初始化不会进行**窄化类型转换 narrowing conversion**，比如从 double 到 int 和从 int 到 char，而 `=` 会进行隐式类型转换。

所有内置类型都可以省略初始化语句，此时变量会被隐式初始化为其默认值，例如（**空初始化列表**也表示使用默认值）：  

``` C++
int a; // 等同于 int a {}; , a 变为 0
double b; // 等同于 double b {}; ，b 变为 0.0
char* p; // 等同于 char* p {}; ，p 变为 nullptr
vector<int> vec; // 等同于 vector<int> vec {}; ，vec 变为空向量容器
string s; // 等同于 string s {}; ，s 变为 ""
```

用户自定义类型默认情况下不会被初始化，除非它们是带有默认构造函数的用户自定义类型，例如上面的 string 和 vector。

# 第七章 函数
## 声明
函数**声明 declaration** 告诉编译器函数的名称、返回类型和参数，例如：  

``` C++
Elem∗ next_elem(); // 返回 Elem 指针
void exit(int);
double sqrt(double);
```

函数声明可以包含<u>参数的名称</u>，但除非在函数声明的同时定义函数体，否则编译器会忽略函数声明中参数的名称，例如：  

``` C++
int max(int num1, int num2);
int max(int, int);
```

<u>函数的类型</u>由它的返回值类型和参数决定，例如：  

``` C++
double get(const vector<double>& vec, int index); // 函数类型为：double(const vector<double>&,int)
```

函数可以是类成员，对于**成员函数 member function** 来说，类名也是函数类型的一部分：  

``` C++
char& String::operator[](int index); // 函数类型为：char& String::(int)
```

可以申明多个相同名称但具有不太参数类型的函数，称为**函数重载 function overloading**：  

``` C++
void print(int);
void print(double);
void print(string);
```

## 定义
函数**定义 definition** 包含了函数的函数体，例如：  

``` C++
void swap(int*, int*); // 声明

void swap(int* p, int* q) // 定义
{
    int t = *p;
    *p = *q;
    *q = t;
}
```
