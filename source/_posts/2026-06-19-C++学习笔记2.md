# 第八章 类的基本概念
**类 class** 是用户自定义类型，由一组**成员 member** 组成，成员包括**数据成员 data member** 和**成员函数 member function**。一个简单的类定义如下：  

``` C++
class MyClass {       
public:                    // 访问修饰符 access specifier
    int myNum;             // 数据成员 data member 
    std::string myString;  // 数据成员 data member 
};
```

定义类之后就可以创建它的类对象了，对象可以分配在栈上也可以在堆上，对于普通对象使用 `.` 运算符访问成员，对于指针对象使用 `->` 运算符访问成员：  

``` C++
int main() {
    MyClass class1; // 栈上分配
    MyClass* class2 = new MyClass; // 堆上分配
    
    class1.myNum = 5;
    class2->myNum = 5;
    std::cout << class1.myNum << std::endl; // 输出 5
    std::cout << class2->myNum << std::endl; // 输出 5
    delete class2;
}
```

## 类内初始化器
C++ 11 引入了**类内初始化器 in-class initializer**，或称为**默认成员初始化器 default member initializer**，它允许你在声明成员变量时直接为其指定默认值，例如：  

``` C++
class MyClass {
    int a = 10;          // 使用 = 
    double b{3.14};      // 使用 {}
    std::string s = "hello";
    // int c(5);         // 错误！不能使用圆括号，会被解析为函数声明
};
```

注意，类内初始化器的实际初始化顺序遵循声明顺序。

## 类访问修饰符
**访问修饰符 access specifier** 主要是用于实现数据**封装 encapsulation**，C++ 提供了三种访问修饰符：  
①公有 public：成员可以从类外部访问；  
②私有 private：成员不能从类外部访问；  
③受保护 protected：成员不能从类外部访问，但其派生类可以访问。

默认情况下，如果不显式指定访问修饰符，类的成员都是 private，例如：  

``` C++
class A { int x; };  // x 是 private，外部不能访问
```

## 成员函数
一般来说，类的定义分为两个部分：类声明和类成员函数定义。类成员函数定义可以在类内部，也可以在类外部。但是通常情况下，建议将类声明放在头文件 <kbd>.h</kbd> 文件当中，类成员函数定义放在 <kbd>.cpp</kbd> 文件当中。

### 类内部定义

``` C++
class MyClass {
public:
    void myMethod() {
        std::cout << "Hello World!" << std::endl;
    }
};
```

在类内部定义的成员函数是隐式内联 inline 的。在上一章函数章节有提过 inline 的作用是把代码复制粘贴到调用处，让程序跑得更快，但是对于现在的编译器来说，是否内联优化几乎完全取决于编译器。而这里的内联，主要是解决一个函数被定义在多个 <kbd>.cpp</kbd> 文件当中从而导致链接时重复定义的问题。比如，如果成员函数定义在头文件 <kbd>.h</kbd> 里，且被多个 <kbd>.cpp</kbd> 文件包含，必须让它隐式内联（写在类内部）或显式加 inline（写在类外部），否则链接器会报错说函数重定义。

### 类外部定义
成员函数若定义在类外部，则需要作用域解析运算符 `::` 来标识函数所属的类：  

``` C++
class MyClass { // 建议放在 .h 文件当中
public:
    void myMethod();
};

void MyClass::myMethod() { // 建议放在 .cpp 文件当中，若放在 .h 文件中建议加 inline
    std::cout << "Hello World!" << std::endl;
}
```

## 构造函数
**构造函数 constructor** 是一种特殊的成员函数，常用于初始化类对象的数据成员，同样可以在类内部或外部定义。构造函数名称与类的名称完全相同，且没有返回类型。一个类可以有多个构造函数，即**构造函数重载 constructor overloading**，只要参数列表不同。我们可以<u>显式</u>地调用构造函数，也可以<u>隐式</u>地调用构造函数，例如：  

``` C++
class Car {
public:
    std::string brand;
    int year;
    Car(std::string x, int y); // 带参数构造函数
};

Car::Car(std::string x, int y) {
    brand = x;
    year = y;
}

int main() {
    Car car1 = Car("Honda", 2019); // 显式调用构造函数
    Car car2("Toyota", 2020); // 隐式调用构造函数
    Car* car3 = new Car("Ford", 2018); // 显式调用构造函数

    Car car4{"Nissan", 2026}; // 直接列表初始化
    Car car5 = Car{"Tesla", 2021}; // 列表初始化 + 复制初始化，C++ 17 后只调用一次构造函数
    Car car6 = {"Mercedes", 2024}; // 复制列表初始化，C++ 17 后只调用一次构造函数
}
```

### 默认构造函数
**默认构造函数 default constructor** 指不需要提供任何实参就可以调用的构造函数，包括两种形式：无参构造函数以及全缺省构造函数（所有参数都有默认值），且每个类最多可以有一个默认构造函数。

``` C++
class Example {
public:
    Example() { /* ... */ } // 1. 无参构造函数
    Example(int a = 0, double b = 1.0) { /* ... */ } // 2. 全缺省构造函数（所有参数都有默认值）
};
```

如果类没有显式地定义任何构造函数，编译器就会为我们隐式地定义一个默认构造函数，又被称为**合成的默认构造函数 synthesized default constructor**，其效果等同于一个函数体和参数列表都为空的默认构造函数。默认构造函数在以下两种初始化场景中会被自动调用：①**默认初始化 default initialization**，例如 `MyClass obj;` 或使用 `new MyClass;`，此时内置类型成员（如 int、指针等）的值是未定义的，因为内置类型成员本身若默认初始化，编译器不会为它们赋一个初始值，除非作为静态存储的变量时被初始化为 0；②**值初始化 value initialization**，例如 `MyClass obj{};` 或使用 `new MyClass();`，此时内置类型会被初始化为默认值 0 或 nullptr。但是，数据成员若有类内初始化器，上述两种形式都会为成员初始化为类内初始化器中的默认值。

另外，有时候编译器不能为某些类合成默认的构造函数，比如类中有未初始化的引用类型的成员变量，或者包含一个其他类类型的成员且这个成员的类型没有默认构造函数，亦或者含有未初始化的 const 成员等等。

### 构造函数初始化列表
推荐使用**构造函数初始化列表 constructor initialize list** 来初始化数据成员，它也称为**成员初始化列表 member initializer list**。初始化列表位于构造函数的参数列表之后，函数体之前，以冒号 `:` 开头，多个成员用逗号分隔，例如：  

``` C++
class Entity {
private:
    int& a;
    const int b;
    std::string name;
public:
    Entity(int& x, int y, const std::string& n) : a(x), b(y), name(n) {
        // 构造函数函数体
    }
};
```

有些成员变量必须放在初始化列表中初始化，否则编译报错，比如：  
①引用类型 `T&`：引用必须在定义时绑定对象，且不可修改；  
②const 成员：常量在声明后不能赋值，只能在初始化阶段设值；  
③没有默认构造函数的类对象：如果成员是一个类，且该类没有提供无参构造函数，你必须在初始化列表里传入构造参数。  

``` C++
class Inner {
public:
    Inner(int val) {} // 有参构造函数，无默认构造函数
};

class MyClass {
private:
    const int c;
    int& ref;
    Inner in;
public:
    // 错误！执行到构造函数体 {} 时，这些成员已经尝试使用默认构造，但 const / 引用未绑定，Inner 无默认构造
    MyClass(int val, int& r) {
        c = val;    // 错误
        ref = r;    // 错误
        in = Inner(10); // 错误（虽然看起来像，但 in 已经构造完了）
    }

    // 正确
    MyClass(int val, int& r) : c(val), ref(r), in(10) {}
};
```

在构造函数体内赋值跟使用初始化列表具有本质区别：<u>在构造函数体内赋值，编译器会先调用成员的默认构造函数创建临时对象，然后再调用赋值运算符覆盖值</u>，这产生了两次函数调用。而使用初始化列表，编译器直接初始化成员对象，只发生一次构造。

``` C++
class Student {
private:
    std::string name;
public:
    Student(const std::string& s) { // 方式一：低效 (先默认构造空string，再赋值)
        name = s;
    }
    
    Student(const std::string& s) : name(s) { } // 方式二：高效 (直接拷贝构造)
};
```

另外，初始化列表的执行顺序，完全取决于成员变量在类中的声明顺序，而不是初始化列表中的书写顺序。<u>建议书写时，永远让初始化列表的顺序与类中声明的顺序严格一致</u>，并且绝对不要用尚未初始化的成员去初始化另一个成员。

### 委托构造函数
C++ 11 引入了**委托构造函数 delegating constructor**，它允许一个构造函数在初始化列表中调用同一个类的另一个构造函数，例如：  

``` C++
class Date {
    int d, m, y;
public:
    Date(int day, int month, int year) : d(day), m(month), y(year) {}
    Date() : Date(1, 1, 1970) {} // 委托构造函数
    Date(int day) : Date(day, 1, 1970) {} // 委托构造函数
};
```

在委托构造函数的初始化列表中只能有这一个委托调用，不能同时有其他成员初始化器。当一个构造函数委托给另一个构造函数时，受委托的构造函数的初始化列表和函数体被依次执行，然后再执行当前构造函数自身的函数体。

### 转换构造函数
如果构造函数可用单个实参调用（只有一个参数，或者有多个参数但除第一个外均有默认值），则它实际上定义了转换为当前类类型的隐式转换机制，称为**转换构造函数 converting constructor**，例如：  

``` C++
class MyString {
private:
    std::string data;
public:
    // 转换构造函数：允许从 const char* 隐式转换为 MyString
    MyString(const char* str) : data(str) { 
        std::cout << "Converting constructor called" << std::endl; 
    }
};

void print(MyString s) {
    // ...
}

int main() {
    print("Hello World"); // 隐式转换，自动调用 MyString(const char*) 进行转换
    MyString s = "C++"; // 隐式转换，等价于 MyString s("C++");
}
```

由于，所有单参数构造函数都是转换构造函数，会导致意外转换，C++ 提供了 `explicit` 关键字，强制要求必须使用直接初始化或显式转换：  

``` C++
class SafeString {
private:
    std::string data;
public:
    explicit SafeString(const char* str) : data(str) { }
};

void log(SafeString s) { /*...*/ }

int main() {
    log("error"); // 错误：无法隐式转换
    SafeString s1("OK"); // 正确：直接初始化
    SafeString s2 = "OK"; // 错误：explicit 禁止复制初始化
    SafeString s3 = SafeString("OK"); // 正确：显式构造后拷贝
    log(SafeString("error")); // 正确：显式构造临时对象
    SafeString s4 = static_cast<SafeString>("OK"); // 正确：static_cast 显式转换
    SafeString s5 = (SafeString) "OK"; // 正确：C 风格强制转换
}
```

## 友元
**友元 friend** 允许外部函数或外部类访问当前类的 private 和 protected 成员，但因为会破坏封装性，建议少用为妙。友元的声明只能出现在类的内部，且不受 public / private 访问修饰符的影响。

①友元函数，即让全局函数能够访问类的私有成员：  

``` C++
class Box {
private:
    double width;
public:
    Box(double w) : width(w) {}
    
    friend void printWidth(Box& b); // 声明全局函数 printWidth 为友元
};

void printWidth(Box& b) { // 注意没有作用域限定符 ::
    std::cout << "Width = " << b.width << std::endl; // 可以访问 private 成员 width
}

int main() {
    Box box(10.5);
    printWidth(box);
}
```

②友元类，即整个类作友元，让友元类可以访问当前类的私有成员：  

``` C++
class A {
private:
    int secret = 42;
public:
    friend class B; // 声明 B 类是 A 的友元
};

class B {
public:
    void show(A& a) {
        std::cout << a.secret << std::endl; // B 可以访问 A 的私有成员
    }
};

int main() {
    A a;
    B b;
    b.show(a); // 输出 42
}
```
