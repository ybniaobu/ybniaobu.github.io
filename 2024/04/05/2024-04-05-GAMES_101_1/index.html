GAMES101-图形学入门公开课笔记（一）

发表于2024-04-05|更新于2024-04-23|图形学
|字数总计:7.6k|阅读时长:27分钟|阅读量:

GAMES101 是一门现代计算机图形学入门课程，全名为现代计算机图形学入门，本课程将全面而系统地介绍现代计算机图形学的四大组成部分：①光栅化成像，②几何表示，③光的传播理论，以及④动画与模拟。由加州大学圣芭芭拉分校 UCSB 的闫令琪教授讲解，他的科研成果被直接应用于工业界，如影片《猩球崛起3：终极之战》与《狮子王2019》，以及与 NVIDIA 合作推动了实时光线追踪技术的产生。

b 站视频 BV 号：BV1X7411F744
我觉得这门课更多起到一个抛砖引玉的作用，让我们对图形学的技术有个大概的了解。

第一课 Overview of Computer Graphics

GAMES101 课程的主要内容为：
①Rasterization 光栅化，即将几何图元 geometry primitives（包括线段、直线、射线、平面、多边形等），转换为片元 fragments（像素 pixels）并投影在屏幕上；
②Curves and Meshes，几何相关内容，即如何在电脑中表示几何 geometry；
③Ray Tracing 光线追踪，从相机发射经过每个像素的射线，计算交接 intersection 和着色 shading，直到光线反射到光源；
④Animation / Simulation 包括 key-frame Animation 关键帧动画 和 Mass-spring System 质量弹簧系统（用于 cloth simulation 布料仿真模拟）

第二课 Review of Linear Algebra

向量、矩阵乘法内容都比较基础，不多做记录了，这里只补充一下数学运算定律相关内容：
①向量点乘满足交换律、分配律和结合律：
   - $\,a \cdot b = b \cdot a\,$
   - $\,a \cdot (b + c) = a \cdot b + a \cdot c\,$
   - $\,(ka) \cdot b = a \cdot (kb) = k(a \cdot b)\,$
②向量点乘不满足交换律，满足分配律和结合律：
   - $\,a \times b = - b \times a\,$
   - $\,a \times (b + c) = a \times b + a \times c\,$
   - $\,(ka) \times b = a \times (kb) = k(a \times b)\,$
③矩阵与矩阵乘法不满足交换律，满足分配律和结合律：
   - $\,AB \neq BA\,$
   - $\,(AB)C = A(BC)\,$
   - $\,A(B + C) = AB + AC\,$
   - $\,(A + B)C = AC + BC\,$

齐次坐标相关内容不多做记录了。知道一下 Affine Transformations 仿射变换（仿射映射）就是一个向量空间进行一次线性变换并接上一个平移，变换为另一个向量空间。即 Affine map = linear map + translation。

还有一个关于齐次坐标的要注意一下，因为齐次坐标的向量是 $\,(x, y ,0)^T\,$，坐标点是 $\,(x, y ,1)^T\,$。而向量加上向量得到的是 $\,(x’, y’ ,0)^T\,$ 也为向量，点减去点是 $\,(x’, y’ ,0)^T\,$ 为向量，点加上向量得到的是 $\,(x’, y’ ,1)^T\,$ 为点。这些都符合定义。但是点加上点，却没有意义，这样会得到 $\,(x’, y’ ,2)^T\,$，为了扩充定义，若 $\,w \neq 0 \,$，则在齐次坐标下的 $\,(x, y ,w)^T\,$ 表示的都是点 $\,(x/w, y/w ,1)^T\,$。这样在齐次坐标下，点加上点得到的是两个点的中点。

View Transformation

观察变换其实就是摄像机带着其他模型一起变换，将摄像机放置在原地，面朝着 -z 方向，即《Unity Shader入门精要》提到的观察变换中的方法二。这些内容《Unity Shader入门精要》也有讲过，不再重复说明了。额外提一句，观察变换还可以用如下的理解方式：观察变换其实就是摄像机空间，即摄像机自己的模型空间，转换到世界空间的逆矩阵，而观察变换不考虑缩放，若只考虑向量方向的话，直接转置就是逆矩阵，因为旋转矩阵是正交矩阵。这和冯乐乐提到的观察矩阵的理解的两种方法逻辑是一样的，忘了可以回去看看。

第五课 Rasterization

该课一开始的视口变换 Viewport transformation 不重复说明了。

Rasterizing triangles into pixels

Raster 在德语中就是屏幕、网格的意思，所以 Rasterize 光栅化就是画上屏幕。

三角形是最基本的形状图元，可以组成任意多边形。三角形可以保证平面是平的，同时可以很好的利用重心坐标插值 barycentric interpolation。

光栅化一个三角形的输入是顶点的屏幕坐标，输出则是屏幕上的一堆近似三角形的像素。重点就是判断像素中心点与三角形的位置关系，基本的方法就叫采样 Sampling。

采样 Sampling 指给定一个连续的函数，根据不同的点去获取一个值。所以可以通过采样去离散化 discretize 一个函数。三角形可以理解为一个函数，然后根据像素中心点进行采样，去输出 0 和 1。至于这个判断三角形内外的函数就是利用了三次叉乘来判断（三个叉乘同号则在内部）。

为了减少判断次数，会先根据三角形三个顶点的最大最小 x 和 y 值来定义出一个边界框 bounding box。边界框外的像素一定不在三角形内部。

最终得到的三角形可能会产生锯齿 jaggies，如下图：

锯齿是光栅化图形学一直致力解决的严重问题。锯齿产生的原因是因为采样率对于信号来说是不够高的，从而产生的信号的走样 Aliasing 问题。抗锯齿即反走样 Anti-aliasing。详见下节课内容。

第六课 Rasterization II

Antialiasing

Sampling theory

Sampling Artifacts 在图形学中指一切采样出现的不如意的地方（Errors / Mistakes / Inaccuracies）。走样 Aliasing 的原因本质就是因为信号变化太快（信号频率高），采样效率（采样频率低）跟不上。而反走样 Anti-aliasing 就是在采样之前做一个模糊 blurring，或者叫做前置滤波 Pre-filtering。这个模糊操作让三角形边缘有了颜色过渡，如下图：

接下来说明频率的问题，首先傅里叶变换可以把信号分解到不同频率（时域变频域），而高频率需要更快的采样的原因如下图所示：

高频率信号受到不充分的采样会让得到信号像一个低频率的信号。使用一个采样率采样两种频率的信号，但无法区分它们，就被称为走样 Aliasing。而滤波的目的就是把特定的频率去除掉。

傅立叶变换可以把图片从时域变换到频域，如下图（一般图片都是低频信息多，高频信息少，亮度表示信息，图像频率就是灰度变化的剧烈程度）：

如果我们通过滤波去掉低频信息，就可以得到图像的边缘，如下图的高通滤波 High-pass filter（高频率可以通过）：

如果我们通过滤波去掉高频信息，就可以得到模糊的图像（Blur），如下图的低通滤波 Low-pass filter（低频率可以通过）：

高通锐化，低通模糊。想要更深地学习这方面知识需要学习数字图像处理相关课程（现在更多地图像操作是根据机器学习来操作的）。

滤波 Filtering 就是卷积 Convolution，可以理解为平均的意思。卷积定理指出在时域上的卷积操作等于在频域上的乘积，时域上的乘积等于频域上的卷积。所以我们可以直接选择在时域上做卷积操作，从而可以简化操作。原操作需要三步：先用傅里叶变换把时域转换至频域，再乘上傅里叶变换后的卷积核，最后逆傅里叶变换回时域。如下图：

图中使用的卷积核叫盒子滤波 Box filter，它是一种低通滤波。更大的卷积核（比如 20 × 20 的卷积核）相当于更低频的低通滤波器。

而采样 Sampling 的本质就相当于重复信号在频域上的内容（Repeating Frequency Contents）。而走样 Aliasing 就是因重复的频率太小而导致的在频域上的混合现象。所以反走样 Antialiasing 就是在采样之前过滤掉高频信息来防止频域上的混合现象。（这部分不放出 ppt 的图片了，若需要更深入学习，建议额外了解，这里大致知道个逻辑即可）

所以反走样采样 Antialiased Sampling 的基本方法就是先使用一个像素宽度的盒子滤波（1 pixel-width box filter）进行卷积操作，再在像素中心进行采样。卷积得到一个像素的平均结果是这个三角形占这个像素的面积，如下图：

其他抗锯齿方案请自行了解：
①快速近似抗锯齿 FXAA (Fast Approximate Anti-Aliasing)，基本逻辑是先得到有锯齿的图片，然后进行边缘检测，对边缘进行替换。速度较快，效果就跟四倍 MSAA 十分接近。
②时域\时间\帧间抗锯齿 TAA (Temporal Anti-Aliasing)，思想是服用上一帧的计算结果，将计算量分摊至多帧的超采样，相当于把超采样分布在时间上。
③深度学习超级采样 DLSS (Deep Learning Super Sampling)，它是一种超分辨率 Super Resolution 的方法，可以将低分辨率的图像转换为高分辨率。

Z-Buffering

Painter’s Algorithm 画家算法就是从远处往前画，不断去覆盖帧缓存 framebuffer。这样可以解决物体远近的问题，但它的时间复杂度太高。所以引入了深度缓存 Z-Buffer，即存储每个像素的深度值 z-value。frame buffer stores color values，depth buffer (z-buffer) stores depth。每个像素点距离近的深度值会覆盖远的深度值，即对每个采样的像素点存储最小的深度值。Z-buffer 的复杂度为 O(n)，Z-buffer 还可以利用 GPU 进行更高效的并行计算。

注意：深度缓冲中的深度值是非线性的。忘记了就看之前的文章。

第七课 Shading

Blinn-Phong Reflectance Model I

最基础的着色模型，着色主要计算的就是在某个特定的着色点 shading point 上朝着摄像机方向的反射光，包括环境光 Ambient lighting、漫反射 Diffuse reflection 以及高光 Specular highlights。着色不处理阴影。

①漫反射项 Diffuse Term
光在所有方向上均匀散射。总的来说，摄像机接受到的光（或者能量）跟光线和法线的夹角的余弦值成正比，即 Lambert’s cosine law 兰伯特余弦定律，又称朗伯余弦定律。

另外考虑光照衰减 Light Falloff，定义在距离点光源距离为 1 的强度 intensity 为 I，那么距离 r 的强度即为 $\,I/r^2\,$。

得到 Lambertian Shading 的公式，该着色与观察方向无关：

$\,L_d\,$：diffusely reflected light；$\,k_d\,$：diffuse coefficient (color)；$\,I/r^2\,$：energy arrived at the shading point；$\,max(0,n \cdot l)\,$：energy received by the shading point。

④最后汇总

Shading frequencies

①平面着色 Flat shading：对每个三角形，即每个面，进行一次 shading，每个面有一个法线。渲染频率为面；
②高洛德着色 Gouraud shading，又称顶点着色：对三角形的每个顶点进行 shading，三角形顶点法线由面法线加权平均而得，三角形内部颜色由插值计算而得。渲染频率为顶点；
③冯氏着色 Phong shading，又称像素着色：在每个像素点上计算着色模型，像素的法线根据重心插值 Barycentric interpolation 计算而得。渲染频率为像素。

Mipmap 需要更大的存储开销，但其实也就多了 1/3 的存储量（按面积去摆放看看就明白了）。

接下来要搞定怎样计算一个像素点覆盖多大的纹理的区域。可以考虑将本像素的中心与邻居两个像素的中心分别映射到纹理坐标上，这里两个像素中心之间的距离正好是覆盖的纹理区域。

可以用相邻采样点在纹理坐标的距离的最大值作为一个像素点覆盖的纹理区域的边长（近似正方形）。得到 L 后，这个近似的 $\,L \times L\,$ 大小的区域在 Mipmap 的第 D 层，即 $\,log_2L\,$ 层上一定会对应到一个像素的大小上。所以我们直接在第 D 层去采样那个像素，就可以得到覆盖的纹理区域的平均值。

由于得到的 D 可能非整数，如果采用近似的方式进行计算，可能会出现过渡不太平滑，这是由于在过度的交界处是在两个不同的层级上进行查询的。一个解决方法是三线性插值 Trilinear Interpolation，即在 D 层上做双线性插值，再在 D + 1 层上做双线性插值，再在层与层之间进行插值。

三线性插值 Mipmap 的缺点是会出现 Overblur 现象：

可以靠各向异性过滤 Anisotropic Filtering 部分解决，由于 Mipmap 采用近似正方形的范围查询，但是有时屏幕像素在纹理上的覆盖区域的长宽压缩不一致。各向异性过滤允许在矩形区域的查询。但它也无法解决覆盖区域是斜着的矩阵或者菱形等，若有更高需求可以尝试了解 EWA filtering。老师说各向异性过滤对计算开销不大，对显存开销大，显存高都可以开。

Applications of Textures

在现代的 GPU 中，texture = memory + range query（filtering）。贴图还有很多应用，比如：
①环境映射 Environment Map
假设光照无限远，可以用一个“镜面球”来记录环境光照，展开这个球就是环境贴图。但是这样会存在一个扭曲问题：在球上，随着维度的升高，越靠近“两极”所占的区域越小，因此虽然一个球体能够描述环境光照，但不是均匀的描述。

一个解决方案：立方体贴图 Cube Map，使用六张图记录环境光。

②凹凸/法线贴图 Bump / normal mapping：存储表面法线；
③置换贴图 Displacement mapping：移动顶点，但模型要有足够的顶点；
④三维程序噪声 3D Procedural Noise；
⑤环境光遮蔽贴图 Ambient occlusion texture map；

⑤分形 Fractals
分形的生成是基于一个不断迭代的方程式，生成无限复杂具备自相似结构。

“Explicit” Representations of Geometry

②多边形网格 Polygon Mesh
不用多说了，最常见的几何表达。

Mesh simplification (downsampling)

目标就是减少网格数量，但是维持整体形状。

比如说边坍缩 edge collapsing，把一些边上的两个顶点变成一个点。但是有个问题就是需要衡量哪些边该坍缩，哪些不该坍缩。这里需要引入一个概念：二次误差度量 Quadric Error Metrics，二次误差就是新的顶点应该要最小化它到坍缩之前相关的几个面的距离的平方和。边坍缩应该选择一个最小二次误差的点。

但是坍缩一条边之后，可能有一些边会受之影响。那么就需要快速查询最小的二次度量误差的同时，以较小的代价去更新其他受影响的二次度量误差的值，可以使用优先队列（堆）数据结构。

有兴趣可以去了解 Quadric Error Mesh Simplification 算法。

Mesh regularization (same triangles)

网格规则化，应该就是网格重拓扑、表面平滑之类的算法，老师上课没讲，有兴趣额外了解相关算法。
